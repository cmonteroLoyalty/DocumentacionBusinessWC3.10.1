Batches v3.9.4

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Create a manual batch

    In addition to uploading batch files, customers can now create a batch order manually. Out of the box, the following manual batch type is supported: USA intracompany credit manual batch.

    This feature allows developers to configure a new manual batch type including validations. Another option is to reconfigure the out-of-the-box manual batch type for any specific internal batch payment type.

    The button to upload a batch in the Manage batch payments widget is now a dropdown with two options: Manual batch or Upload batch. Clicking Manual batch opens a new modal that consists of the following steps:

    1. Create a batch order in draft mode

    2. Add, edit, and delete batch payments to a batch order in draft mode

    3. Validate and submit a batch order

    Every entered detail is validated on each step, and the user may only progress after each field is valid.

Cash flow forecasting v3.9.4

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Widgets updates

    • Fix collapsible-cards on view and create settings widget.

    • ItemModel is optional for CFF widgets.

Bug fixes

This release includes the following bug fixes:

  • CMT-1206: Restart accounting sync with ERRORED status to enable dependent syncs without errors to run

    An accounting sync with ERRORED status would not allow dependent syncs without errors to run and complete.

    A sync with ERRORED status now restarts and enables all syncs to complete.

Payments v3.9.4

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Clear button to remove entered or selected beneficiary details

    The Initiate omni payment and Initiate payment wizard widgets now have a Clear button that allows users to remove entire applied beneficiary details without clearing the entered information from From account and Payment details.

  • Ability to show a purpose of payment in the details of a payment template, and ability to apply it during the creation of a payment

    • View Template widget now supports showing the ‘Purpose of payment’ as part of the template details, either in a free-text or PoP code and description format.

    • User can now apply this value in ‘Purpose of payment’ when they create a payment from this template, either via the Manage templates or the template selector.

Products v3.9.4

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Enable users to remask an Account Number which was unmasked

    When navigating to View business account details widget, the business users are now able to remask an account number which was unmasked.

  • Favourite Accounts, Account Picker and Account Overview Widgets support Product Summary View and Product Summary Limited View by default

  • User Preference Restrictions for users with the Product Summary Limited View Business Function

    • Limited Account Overview

      Users with the Product Summary Limited View permission can only view a limited subset of Data attributes associated to an account in the Accounts Overview.

    • Limited Account Details

      Users with the Product Summary Limited View permission can only view a limited subset of Data attributes associated to an account in the Accounts Overview.

    • No Balance History Tab for Limited View

      Users with the Product Summary Limited View permission for a given arrangement will not have the Balance History Tab propagate within the Account Details. This is due to the fact that users with the Limited View permissions are restricted from seeing any balance related information.

    • Restrict use of unmask functionality for Limited View Permissions

      Business users with limited view permissions will not be able to use option mask/unmask/remask Account number in View business account details. This option will be added in a future release.

Bug fixes

This release includes the following bug fixes:

  • MAINT-11555: View business accounts list widget - UI components listed in API docs of widget are not exported as public API (DBBES-13159)

    The UI components listed in the API docs of View business accounts list widget are now exported as public API and can be accessed via export { AccountsOverviewContainerComponent } from '@backbase/business-ang/product-summary This issue was also fixed as a patch for version v3.8.1 - and it is released with patch version v3.8.1-sp.17.

User Self Service v3.9.4

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-11743: Add extension point for error messages in the Manage user security widget.

    Previously, you couldn’t change the error message the widget displayed to explain why a user couldn’t change their password. The widget always displayed the notification "Could not update password. Incorrect login credentials or password policy not met."

    To enable you to change this notification, it has been moved out to the parent container, which allows extension.

Migration

When you migrate from one version of this product to another, best practice is to update incrementally through all minor and major versions in between.

Follow these steps to update an existing deployment:

  1. Add limited view business function to Products widgets

    Change the widget preference with the title Default business function and the name businessFunction to Product Summary,Product Summary Limited View in the following widgets:

    • Select business account: bb-account-picker-widget

    • View business account list: bb-accounts-overview-widget

    • View favorite accounts: bb-product-summary-account-favorites-widget
