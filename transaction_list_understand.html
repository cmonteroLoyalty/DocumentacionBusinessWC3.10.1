Understand Transactions

A financial transaction is an agreement between a buyer and a seller to exchange goods, services or financial instruments. Transactions in Backbase Platform provides an overview and a detailed view of transactions for one or more financial products.

Completed or billed transactions are executed and settled payment orders. This is the final state of a transaction. A pending transaction is an authorized transaction. The balance of a pending transaction is held as unavailable either until the counterparty clears the transaction, or until the hold drops off. Transactions supports both completed and pending transactions.

This topic explains the following about Transactions:

  • Transactions provides the following functionality:

    • List transactions

    • Enrich transactions

    • Export transactions

    • Search transactions

    • Filter transactions

    • Display check images

    • Display geolocation

  • Overview: Widget collections and DBS support

List transactions

This functionality includes the following features:

  • List transactions for one or more product kinds, such as accounts, or cards. By default, transactions are presented to the bank customer with most recent transactions on top of the overview.

  • List transactions with details, for example: amount, booking date, description, reference, type group, type, category, credit/debit indicator, instructed amount, instructed currency, currency exchange rate, counter party name, counter party account number, check serial number, billing status, address, and running balance of transactions.

  • Load transactions incrementally.

  • Configuration options to list pending transactions on top of completed transactions, or mix transactions regardless of their billing status.

  • Configuration options to set the number of transaction records per page, and the number of navigation pages.

  • Display the account number of the user involved in a specific transaction, when more than one account is selected for viewing transactions.

  • Sort the transactions list by specific criteria. By default, the transactions are sorted in the descending order of the booking date.

  • Display different icons for transactions, based on available data. These icons include the following:

    1. Merchant icons

    2. Category icons

    3. Transaction type icons

    4. Icoming and outgoing transaction icons

    The list above represents the order of priority. For example, if there is no data available to display a merchant icon, the fallback is the category icon, and so on.

This functionality interfaces with Transactions using the transaction-manager-client-api.

Enrich transactions

Note
 This section describes an add-on. You may use this add-on only if you have purchased an appropriate license for this add-on based on a signed Product Order Form.

If transaction enrichment is configured in DBS, transactions can be automatically assigned to a category provided by a third-party transaction enrichment provider. If this feature is enabled, the enriched icons are displayed in the transactions list and details views.

Export transactions

Bank customers are able to download and export transactions from the list of transactions. The available formats to export depend on what is supported by the backend. For the formats supported by DBS out of the box, see Transactions.

If the bank customer applied a filter or searched on a keyword in the transactions list, only the filer or search result list is exported. If the bank customer attempts to export a greater number of transactions than the configuration allows, an error message is displayed and the download of the export files does not start.

This functionality interfaces with the Transactions DBS capability using the GET //transaction-manager-client-api/client-api/v2/transactions/export endpoint.

Bank customers can export transactions in CSV, OFX, QFX, QBO, or PDF format. The OFX, QFX, and QBO export formats are only available for checking, savings, and credit card accounts. For more information, see Export transactions.

Filter transactions

Bank customers can filter their transactions list based on a set of criteria, including:

  • Credit/debit indicator

  • Transaction type

  • Date range

  • Amount range

  • Check serial number range

It depends on the backend support which criteria are available. See Transactions for the filter criteria supported by DBS out of the box.

Search transactions

The bank customer can run a free text search with keywords and searchable attributes. Both full and partial matching is supported, for example:

  • If the bank customer searches "Insurance", the system scans all the searchable fields that contain the string Insurance.

  • If the bank customer searches "Insur", the system scans all the searchable fields that contain the string Insur.

  • If the bank customer searches "Car Insurance", the system can behave in several ways depending on the search mode configuration in the service.

The attributes on which users can search depend on the backend support. For the attributes supported by DBS out of the box, see Transactions.

The query includes notes that the bank customer added to transactions previously.

This functionality interfaces with the Transactions DBS capability using the transaction-manager-client-api.

Display check images

If the selected transaction is a check withdrawal or a check deposit, and contains check images, then the images of the front and the back of the cleared check can be displayed in the transaction details as thumbnails. Check images are available to the back customer for download. This functionality does not support images of deposit slips.

This functionality interfaces with the Transactions using the GET //transaction-manager-client-api/client-api/v2/transactions/{transactionId}/check-images endpoint.

Display geolocation

Geolocation is the identification or estimation of the real-world geographic location of an object. If latitude and longitude values are present for a transaction, Transactions can display a map in transaction details which shows the geolocation of that transaction.

This functionality interfaces with Transactions using the transaction-manager-client-api.
