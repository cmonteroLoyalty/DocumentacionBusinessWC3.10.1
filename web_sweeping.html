Web widgets

The web widgets for this capability are:

  • Create sweep

  • Manage sweeps

  • View sweep details

Create sweep

In this widget, the financial institution customer creates a one- or a two-way daily sweep.

The customer clicks the New sweep button in the Manage sweeps widget to create a new sweep, then goes through steps:

  1. Set type of rule.

    The customer defines the following:

    • Sweep name: Any name given by the customer.

    • Sweep type: A one-way sweep or two-way sweep.

    • Sweep condition: The condition which is applied to the target account and, if met, triggers a sweep, which can be:

      • One-way: The customer selects either of the two conditions:

        • is higher than: The balance of the target account exceeds a specified amount (default value) OR

        • is lower than: The balance of the target account is lower than a specified amount

      • Two-way: Both conditions are applied:

        • is lower than: The balance of the target account is lower than a specified amount AND

        • is higher than: The balance of the target account is higher than a specified amount

    • Amount: The threshold that triggers the sweep.

    • Target account: The target account name and IBAN. If IBAN is not available, BBAN is displayed.

    • Swept account: The swept account name and IBAN. If IBAN is not available, BBAN is displayed.

    • Description: An optional additional information about the sweep.

  2. Schedule sweep.

    The customer defines the following:

    • Start date: The current date or a future date.

    • Expiry date: The current date, a future date, or never.

    • Repeat every:

      • Frequency count: The number of times of how often to repeat the sweep.

      • Frequency period : Day, Week, or Month

    • Repeat on: A specific day on which the sweep is repeated. Available only for weekly and monthly frequency periods.

    • At: The exact time when sweeping is done

      • Starting at

      • Ending at

      • Continuing every

        Starting at, Ending at, and Continuing every are available only for daily frequency period, when Repeat during the day is checked.

    To enable intraday sweeping, the bank customer selects Repeat during the day. The setting At then changes to Starting at. If the number of hours is set to 00, only the number of minutes are taken into consideration when repeating the sweep. If the number of hours is set to 01 or greater, both the number of hours and the number of minutes are taken into consideration.

    If the bank customer defines a period at Starting at and Ending at that is shorter than the amount of hours set at Continue every, the sweep is performed only once.

  3. Review sweep.

    The customer reviews the data provided in the previous steps. The customer clicks Submit to confirm the new sweep. Once the sweep is created, the customer is redirected to the Manage sweeps widget, where they see the new sweep listed.

Features

This widget provides the following features for the customer:

  • Create a new sweep.

  • Set properties for the sweep.

Manage sweeps

In the Manage sweeps widget, the bank customer views and manages available sweeps. The customer may also search for sweeps by name and filter sweeps using several criteria.

The information includes the following details:

  • Status: The current sweep life cycle status.

  • Name, ID: Any name given by the customer and the unique sweep identifier in the system.

  • Sweep rule: The condition that triggers the sweep.

  • Target account: The account name and IBAN. If IBAN is not available, BBAN is displayed.

  • Swept account: The account name and IBAN. If IBAN is not available, BBAN is displayed.

  • Execution: The time and frequency of sweeping.

The customer can activate, pause, or cancel the sweep from the list, as well as print detailed information about it.

The filtering criteria include the following:

  • Sweep status: Active, paused, expired, cancelled, rejected, processing, entered

  • Sweep execution rule: Surplus, maintenance, two way

  • Sweep execution time: Daily, weekly, monthly

The customer selects no, one, or multiple values for each filtering criteria. Search and filtering can be applied simultaneously.

Features

This widget provides the following features for the customer:

  • View high-level info about sweeps.

  • Filter sweeps by status, rule, and execution.

  • Search sweeps by sweep name.

  • Open the View sweep details widget.

  • Activate, pause, or cancel sweep.

  • Print sweep details.

View sweep details

In the View sweep details widget, the bank customer views details of an existing sweep.

The information includes the following:

  • Sweep name: Any name given by the customer.

  • Sweep ID: The unique sweep identifier in the system.

  • Username: The name of the customer that created the sweep.

  • Created on: The server date when the sweep was created (local date).

  • Created at: The server time when the sweep was created (local time).

  • Sweep status: The current sweep life cycle status.

  • Sweep rule: The sweep type and amount. Sweep type can be:

    • One-way (surplus or maintenance)

    • Two-way

  • Target account: The account name and IBAN. If IBAN is not available, BBAN is displayed.

  • Swept account: The account name and IBAN. If IBAN is not available, BBAN is displayed.

  • Sweep execution: The time and frequency of sweeping.

  • Sweep description: Optional additional information about the sweep.

  • Start date: The current date or a future date.

  • Expiry date: The current date, a future date, or never.

  • Repeat every:

    • Frequency count: The number of times of how often to repeat the sweep.

    • Frequency period: Day, week, or month.

  • Repeat on: The specific days on which the sweep is repeated. Available only for weekly and monthly frequency periods.

  • At: The exact time when sweeping is done. If Repeat during the day is checked, then At changes to:

    • Starting at

    • Ending at

    • Continuing every

Features

This widget provides the following features for the customer:

  • View sweep details.

  • Activate, pause, or cancel sweep.

  • Print sweep details.
