View favorite accounts

  • Selector name

  • Required imports

  • Extension slots

  • API

  • Implementation example

  • Preferences

Selector name

This widget has the following selector name: bb-product-summary-account-favorites-widget

Required imports

The following imports are required to set up and customize this widget:

Widget module import

Import the following modules into the module of your wrapper widget:

import { ProductSummaryAccountFavoritesWidgetModule } from '@backbase/product-summary-account-favorites-widget-ang';

Extension slots

This widget has the following extension slots:

  1. bbAccountFavoritesWidgetCustomizable

  2. bbAccountFavoriteCardsCustomizable

bbAccountFavoritesWidgetCustomizable

This extension slot includes the following default template:

<ng-template
  bbAccountFavoritesWidgetCustomizable
  let-hostRef
  let-accounts="context.accounts"
  let-loading="context.loading"
  let-error="context.error"
  let-allowedAccountNavigationTypes="context.allowedAccountNavigationTypes"
  let-isEditAllowed="context.isEditAllowed"
>
  <bb-product-summary-loading
    [isLoading]="loading"
    classes="bb-account-favorites-loader"
    text="Loading Accounts"
    i18n-text="Loading Accounts@@favorite-accounts-container.loading-text"
  >
    <ng-container *ngIf="!error; else errorTemplate">
      <ng-container *ngIf="accounts && accounts.items.length > 0">
        <div class="bb-block bb-block--md bb-account-favorites">

          <bb-header-ui
            headingClasses="bb-block bb-block--md bb-account-favorites__title"
            headingType="h2"
            i18n-heading="Favorites|Favorite Accounts header@@favorite-accounts-container.header"
            heading="Favorites ({{accounts.items.length}})"></bb-header-ui>

          <bb-accounts-favorite-cards
            *ngIf="allowedAccountNavigationTypes"
            [accounts]="accounts.items"
            [allowedNavigations]="allowedAccountNavigationTypes"
            [canToggleFavorite]="isEditAllowed"
            (selectedAccount)="hostRef.onSelectedAccount($event);"
            (favorite)="hostRef.toggleAccountFavorite($event)"
            (viewDetails)="hostRef.onNavigateToDetails($event)"
            (viewTransactions)="hostRef.onNavigateToTransactions($event)"
            (viewStatements)="hostRef.onNavigateToStatements($event)"
            (viewBalanceHistory)="hostRef.onNavigateToBalanceHistory($event)"
            (newPayment)="hostRef.onMakePayment($event)"
          ></bb-accounts-favorite-cards>
        </div>
      </ng-container>
    </ng-container>
  </bb-product-summary-loading>

  <!-- Error template -->
  <ng-template #errorTemplate>
    <bb-empty-state-ui
      class="m-5 d-flex flex-column"
      [title]="hostRef.processErrorMessages(error?.messages)"
      iconModifier="error">
    </bb-empty-state-ui>
  </ng-template>
</ng-template>

This extension slot consumes the following object properties from its custom context:

Object property Description

accounts: Accounts

Accounts list.

loading: boolean

Indicates if accounts loading is in progress.

error: AccountsOverviewError

Contains accounts fetching errors.

allowedAccountNavigationTypes: AccountNavigationType[]

Allowed navigations from account, based on user permissions.

isEditAllowed: boolean

Defines if edit actions allowed.

bbAccountFavoriteCardsCustomizable

This extension slot includes the following default template:

<ng-template
  bbAccountFavoriteCardsCustomizable
  let-hostRef
  let-accounts="context.accounts">
  <div class="bb-stack bb-account-favorites__card-list">
    <div class="bb-stack__item bb-account-favorites__card"
         *ngFor="let item of accounts">
      <div class="card"
           [class.bb-accounts--favorite]="hostRef.isFavorite(item)"
           [ngClass]="'bb-accounts-' + hostRef.toKebabCase(item.productKindName)"
      >
        <bb-product-item-card
          (itemSelect)="hostRef.onSelectedAccount($event)"
          (favorite)="hostRef.onFavorite($event)"
          (viewTransactions)="hostRef.onViewTransactions($event)"
          (viewStatements)="hostRef.onViewStatements($event)"
          (viewBalanceHistory)="hostRef.onViewBalanceHistory($event)"
          (newPayment)="hostRef.onNewPayment($event)"
          [allowedNavigations]="hostRef.allowedNavigations"
          [canToggleFavorite]="hostRef.canToggleFavorite"
          [isFavorite]="hostRef.isFavorite(item)"
          [item]="item"
          class="card-body"
        >
        </bb-product-item-card>
      </div>
    </div>
  </div>
</ng-template>

This extension slot consumes the following object properties from its custom context:

Object property Description

accounts: ProductsummaryItem[] | undefined

Accounts list.

Implementation example

The following examples show the implementation of this widget:

Wrapper widget module

import { NgModule } from '@angular/core';
import { CommonModule } from '@angular/common';
import { BackbaseCoreModule } from '@backbase/foundation-ang/core';
import { ProductSummaryAccountFavoritesWidgetModule } from '@backbase/product-summary-account-favorites-widget-ang';
import { ProductSummaryAccountFavoritesWidgetExtendedComponent } from './product-summary-account-favorites-widget-extended.component';

@NgModule({
  declarations: [ProductSummaryAccountFavoritesWidgetExtendedComponent],
  imports: [
    CommonModule,
    BackbaseCoreModule.withConfig({
      classMap: { ProductSummaryAccountFavoritesWidgetExtendedComponent },
    }),
    ProductSummaryAccountFavoritesWidgetModule,
  ],
  exports: [ProductSummaryAccountFavoritesWidgetModule, ProductSummaryAccountFavoritesWidgetExtendedComponent],
})
export class ProductSummaryAccountFavoritesWidgetExtendedModule {}

Wrapper widget component

import { Component, EventEmitter, Output } from '@angular/core';

@Component({
  selector: 'bb-product-summary-account-favorites-widget-extended',
  templateUrl: './product-summary-account-favorites-widget-extended.component.html',
})
export class ProductSummaryAccountFavoritesWidgetExtendedComponent {
  @Output() selectedAccount = new EventEmitter<string>();
  @Output() navigateToDetails = new EventEmitter<string>();
  @Output() makePayment = new EventEmitter<string>();
  @Output() navigateToTransactions = new EventEmitter<string>();
  @Output() navigateToStatements = new EventEmitter<string>();
  @Output() navigateToBalanceHistory = new EventEmitter<string>();
}

Custom template

<bb-product-summary-account-favorites-widget

></bb-product-summary-account-favorites-widget>

Default business function

In most cases, you should not change the default values.

The widget uses the resource, business function(s) and privilege in the following call to arrangement manager:

GET https://ips_host:ips_port/gateway/api/arrangement-manager/client-api/v2/productsummary/context/arrangements?businessFunction=Product%20Summary,Product%20Summary%20Limited%20View&resourceName=Product%20Summary&privilege=view

The arrangement-manager checks with Access Control whether the user has the privilege for any of the specified business functions. Customers with privileges for the Product Summary Limited View business function only see a limited set of keys for an arrangement, see Limited view.

Access Control

A permission is the combination of a business function and a single privilege granted to a user on a given resource type. Privileges are enforced by Digital Banking Services. Widget behavior is affected by the permissions of the user accessing the widget.

The following table describes how the widget behavior changes depending on the permissions of the user:

Resource

Business function

Privilege

Widget behavior

Product Summary

Product Summary

view

The current user views a list of accounts set as a favorite.

If the user does not have the permission to view any of the two business functions, the widget displays an empty list.

edit
Untitled

The current user is able to:

  • Set an account as favorite.

  • Remove an account from favorites.

If the user does not have the permission to edit, these options are unavailable.

Product Summary Limited View

view

The current user views a list of accounts, that are set as a favorite, with limited data. See Limited view.

If the user does not have the permission to view any of the two business functions, the widget displays an empty list.

Untitled

The current user is able to:

  • Set an account as favorite.

  • Remove an account from favorites.
