Actions v3.6.2

This release note explains changes to this capability.

New features

This release provides the following new feature:

  • Updated widget design

    The loading state has been added to the toggle switcher in the Configure business approval notification settings.

Authentication v3.6.2

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Set an action to perform after login

    The Log in user widget now provides the Output Login Action preference as an alternative to the Login Success Redirect To preference. Use this new preference to configure the widget to perform an action after a user successfully logs in.

    A developer sets up this action, either by creating an output handler or wrapping a widget extension. For example, the developer can create an output handler that selects the context for the user if that user is assigned to only one service agreement.

Cash flow forecasting v3.6.2

This release note explains changes to this capability.

New features

This release provides the following new features:

  • View the due amount

    When viewing receivables and payables, the user can now view the outstanding amount to be paid on a partially paid payment order.

  • View the time of the last update

    When viewing receivables and payables, the interface now shows the user a timestamp of the last time Cash Flow Forecasting was synced with the accounting software.

Known issues

This release has the following known issues:

  • Discovered in v3.6.2

    • CME-673: Error in the console when trying to use search.

      When using Ubuntu, the Manage payables and View receivables crashes when a user attempts a search. On other operating systems and browsers, the widget does not crash but throws an error in the console.

      There is no current workaround, and the team is currently investigating.

New features

This release provides the following new features:

  • Mark all notifications as read

    The notification recipient can now mark all notifications as read in the Manage and count notifications widget.

Payments v3.6.2

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Show payment summary for bulk approvals

    With this feature, customers see a summary of payments selected in bulk for approval. The summary is grouped per currency, showing the number of payments and the aggregated amount. It also shows account name and account number, if all payments in a group are from the same account. If there are payments from multiple accounts in a group, it shows 'Multiple Accounts'.

  • Add approval comment for bulk approvals

    Customers can now add an approval comment on the payment orders they are approving or rejecting in bulk. Adding an approval comment is optional and it has a maximum length of 140 characters.

Products v3.6.2

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Add Pagination Type property to View business account list widget

    The bank user is now able to update Preferences section in Experience Manager for View business account list and configure the pagination type to:

    • Default - switch from pagination to Load more option when screen resolution is in Responsive view.

    • Pagination - paginate the number of records on the screen regardless of the screen resolution.

    • Load More - apply load more to the page, regardless of the screen resolution.

  • Add Pagination Type property to View business account balance history widget

    The bank user is now able to update Preferences section in Experience Manager for View business account balance history and configure the pagination type to:

    • Default - switch from pagination to Load more option when screen resolution is in Responsive view.

    • Pagination - paginate the number of records on the screen regardless of the screen resolution.

    • Load More - apply load more to the page, regardless of the screen resolution.

  • Show All Accounts label only in case user has selected favorite accounts

    The bank customer is now able to view All accounts label in View business account list widget only in case user has favorite accounts selected.

  • Option to retrieve and show a specific product kind

    The bank user is now able to update Preferences section in Experience Manager for View business account list and filter for the Account Type that will be displayed in the overview list page.

  • Migrate View business banking balance history widget to Theme 2.0

    View business banking balance history widget is now migrated to Theme 2.0. The widget was visually upgraded to offer an improved and simplified user experience for bank customers. The balance history charts have been updated and three new extension slots have been introduced for this widget:

    • bbBalanceHistoryLineChartCustomizable - used for the whole balance history chart.

    • bbBalanceHistoryLineChartSeriesTooltipCustomizable - used to display the tooltip for positive and negative series.

    • bbBalanceHistoryLineChartTooltipCustomizable - used to show the tooltip for a single point in the graph.

  • Pin favorite accounts in the Account list business banking app

    A bank user can now pin listed accounts as favorites. The account is pinned to the top of the list for quick access. Once pinned, the bank user can view the account’s balance, currency, and balance trend. A maximum of six favorite accounts are shown at the top of the account list. The way a user pins an account as a favorite is platform specific:

    • iOs: Swiping the listed account to the left reveals the option to select or deselect that account as a favorite.

    • Android: Long-pressing a listed account reveals the option to select or deselect that account as a favorite.

Bug fixes

This release includes the following bug fixes:

  • PRSUM-6489: Update list of preferences for Select business account widget

    Preferences section for Select business account widget is now updated.

Sweeping v3.6.2

This release note explains changes to this capability.

Known issues

This release has the following known issue:

  • Discovered in v3.6.2

    • WM-2132: Error message appears after manually inputting valid date.

      This issue affects the Create sweep widget when set to French locale. An error message stating Please check your input appears, even when the user inputs a valid date.

      To work around this issue, change the locale from French to English.

Transactions v3.6.2

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Ability to view the geolocation and address of a transaction

    The View transactions table widget now provides the ability to view the geolocation of a transaction in a map along with the open address so that bank customers easily identify their transactions.

Known issues

This release has the following known issue:

  • Discovered in v3.6.2

    • WM-2132: Error message appears after manually inputting valid date.

      This issue affects the View transactions table widget when set to French locale. An error message stating "Please check your input" appears, even when the user inputs a valid date.

      To work around this issue, change the locale from French to English.

New features

This release provides the following new features:

    The Manage user security widget has been added to Widget Collection Business Banking. This widget enables a user to change their password.
