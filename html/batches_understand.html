Understand Batches

With Batches, business users create and send multiple payments in one go instead of creating each payment individually, while staying in control of all batches and related payments with an actionable overview.

Batches provides the following functionality:

  • Upload batches

  • Create a batch manually

  • Manage batches

Upload batches

Business users create batch payments by uploading a file. While uploading a file, the following features are available:

  • Browse for files on your device to upload.

  • Select a batch file type to upload. The following types of batch files are supported by default:

    • SEPA credit CSV file

    • NACHA ACH credit fixed-length file

      Both balanced and unbalanced are supported. A balanced ACH credit batch file contains per batch one debit payment that should match the total sum of the credit payments in the batch.

    • NACHA balanced ACH debit fixed-length file

    You can find more information on the supported file formats in the Payments reference in the DBS documentation.

  • Select multiple files to upload at the same time.

  • Upload the selected file or files asynchronously.

  • Track the progress of the upload.

  • Cancel an upload in progress.

  • Receive a notification when the upload is completed.

  • Receive a notification when the upload failed.

  • View a paginated list of uploaded batch files with status.

Create a batch manually

Business users create batch payments via a user-friendly wizard. The wizard has these features:

  • Create a batch order of batch type USA Intracompany

  • Add, edit, and delete payments to the batch order

  • Review before submitting the batch order

Manage batches

Business users have the following options to manage batches:

  • View a paginated list of uploaded batches with status.

  • Select a batch to view the payments included.

  • Select a batch to view more information about it.

  • Filter batches by amount, execution date, and status.

Batches integrates with Approvals and Limits in order to implement an approval flow for batch payments. Business users with permission can perform the following actions:

  • Delete a batch.

  • Cancel a batch

  • Approve a batch.

  • Reject a batch.

  • Receive a notification if there is a limit breach.
