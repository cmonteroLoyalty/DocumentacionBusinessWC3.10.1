All capabilities v3.8.0

This release note lists the changes to all Business Banking Widget Collection capabilities in this release.

Breaking changes

This release includes the following breaking changes:

  • Support ceases for Angular 8. All Angular widgets, journeys, and libraries are upgraded to Angular 10. See Update to @backbase/foundation-ang v6 for migration recommendations.

Known issues

This release has the following known issue:

  • Discovered in v3.8.0

    • No support for translation extraction with Ivy enabled on app level. This issue is related to this limitation described in the Angular docs: Extract the source language file.

Account statements v3.8.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: New preferences for the View and download business account statements widget

    The following new preferences configurable in Experience Manager have been added for the View and download account statements widget:

    • Relative filter date-picker limit (months)

      When this preference is set, the number of months between the current month and the month limit selected by the bank is calculated. When using the date filter, the bank customer is not allowed to pick up a date (period) beyond the given number of months.

    • Absolute filter date-picker limit (yyyy-mm-dd)

      When this preference is set, the time period between the current date and the date set by the bank customer is calculated. When using the date filter, the bank customer is not allowed to pick up a date (period) beyond the given number of months. If this preference is set, it overrides the value set in the Relative filter date-picker limit (months).

Batches v3.8.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Incorporated ACH-specific batch details and column details changes in the list of batches and the batch details view

    Based on the batch type, such as a SEPA batch or an ACH batch, different details are displayed in both the list of batches and batch details views. This includes the following:

    • For both views: Account details are shown as either IBAN or Routing number + Account number based upon the batch type.

    • For batch list: The Transfers, CCY, and Amount columns are merged into one column and renamed to Credits.

For specifc batch details: CompanyID, Company Name, and SEC type are additionally shown for ACH batches. The CCY and Amount columns are merged into one column and renamed to Credits.

Cash Flow Forecasting v3.8.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: New features in viewing receivables/payables

    • Added the Expected date data field and the possibility to modify it

    • Added grid sorting by issue date, due date, expected date (default), and customer/supplier

  • Web: Filtering options in receivables/payables

    Added generic filtering options: Pending, All, and Paid.

  • Web: Create forecast across all currencies and accounts

  • Web: New features in viewing the forecast

    • The forecast calculation is now based on the expected date in payables and receivables instead of the due date.

    • Added the ability to view weekly and monthly cadences.

    • Added icons for the accounting category.

    • Added styles for the current cadence (borders).

    • Changed the color scheme, historical cadences are now grayed out.

    • Added indication of the forecast currency.

Contacts v3.8.0

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-10551: Contact details notifications are not configurable

    It was not possible to customize the timeout for notifications about contact details. To resolve this, a configurable property notificationDismissTime was added to the Manage contact details widget. This configuration can be changed through Experiene Manager directly and accepts an int value for timeout in miliseconds. The bb-contact-details-container extension slot now also includes the notificationDismissTime input property where this value can be passed through customization.

  • MAINT-10462: Contact’s bbIbanValidatorUi doesn’t support all countries

    Not all countries were supported for saving IBANs as a contact. A valid IBAN for some countries was returned with an error by validator. This was resolved by adding a comprehensive list of IBAN-supported countries.

Notifications v3.8.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Navigation from the payment rejected notification

    The notification about a rejected payment in the Notification badge and View sticky and popup notifications widgets is now clickable and redirects the recipient to the relevant Payment widget.

  • Web: Web notification channel no longer hardcoded

    The previous implementation of the Manage business approval notification preferences widget had web channel hardcoded. This was removed, so that default channels are now taken from configured defaults in action recipe specifications.

Payments v3.8.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: New config for International Transfer - beta (sample code)

    New packaged configuration is available for initiating an International SWIFT Transfer, using the Initiate omni payment widget.

    This configuration features:

    • Ability to show allowed currencies only

    • Ability to show allowed countries only

    • Ability to show exchange rate (indicative) between debit account currency and payment amount currency

    • Ability to show delivery options (settlement time, cut off times, transfer fees, urgent transfer allowed)

    • Ability to show fee sharing options (OUR, BEN, SHA)

    • Ability to route the payment via domestic or foreign correspondent and intermediary banks

    • Ability to show a summary of amounts and charges

      Note
      		 This is a beta (sample code) release. This feature may, unless otherwise authorized by Backbase, be used for testing purposes only, and is provided on an "AS IS" basis without support of any kind. You should not use this feature in a production environment.

  • Web: Save as a template only if allowed

    The button to Save as a template for a submitted payment in the Initiate omni payment widget is shown only if the user has dedicated permission to create a template.

  • Web: New Payment wizard widget - beta (sample code)

    In contrast to the Initiate omni payment widget, where you can define separate entry points for each payment type, Payment wizard offers a single unified UI that guides the user to the correct payment type based on the initial inputs. The widget also provides possible delivery options for each possible payment type, such as the instruction priority, the transfer fee, the fee sharing options, the execution timeframe, and the cutoff time. This helps the user to make an informed choice, while maintaining the control over the payment method. You can reuse the Initiate omni payment widget configurations for the supported payment types or methods to power the Payment wizard.

    Note
    		 This is a beta (sample code) release. This feature may, unless otherwise authorized by Backbase, be used for testing purposes only, and is provided on an "AS IS" basis without support of any kind. You should not use this feature in a production environment.

  • Web: Changes in styles for omni configs

    Utility classes for padding and margin are removed from the cssClasses option in the form field configs of the Initiate omni payment widget. This makes existing payment configs fully compatible for using within the Payment wizard widget.

Deprecations

This release includes the following deprecations:

  • In v3.9.0:

    • The Initiate payment widget will be removed. It is now replaced with Initiate omni payment widget.

Transactions v3.8.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Ability to add notes

    Now the bank customer can add notes to their transactions using the editor in the transaction details of the View transactions table widget. Editing and removal of notes are also supported, along with the ability to search transactions notes.

  • Web: Exporting transactions in QFX format

    Now the bank customer can export their list of transactions in QFX format, so that they can easily use the accounting software provider Quicken.

Migration

When you migrate from one version of this product to another, best practice is to update incrementally through all minor and major versions in between.

  1. Support ceases for Angular 8. All Angular widgets, journeys, and libraries are upgraded to Angular 10

    See Update to @backbase/foundation-ang v6 for migration recommendations. See the Troubleshooting Angular 10 upgrade guide to see solutions to common problems when upgrading.
