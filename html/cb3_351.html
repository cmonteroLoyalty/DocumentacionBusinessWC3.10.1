Batches v3.5.1

This release note explains the changes to this capability.

New features

The new features in this release are:

  • Ability to delete batch orders

    The initiator of the batch can delete the batch in the Manage batch payments widget while it’s still in the ENTERED state, which means that it has not been fully approved or sent to the core banking system yet. This allows the uploader to remove batches that, for example, have been uploaded by mistake.

Bug fixes

The following issues have been fixed in this release:

  • CB-4175: Batches: The tab order and the voice-over accessibility issue in the title of the duplicate file validation screen.

  • CB-4178: Batches: The voice-over accessibility issue of the notification message.

  • CB-4318: Batches: The voice-over accessibility issue in the "Select batch file type to upload" title during batch upload.

FX Trading v3.5.1

This release note explains the changes to this capability.

New features

The new features in this release are:

  • The FX Rates Table widget renamed to Forex.

    The widget provides the following new features:

    • Access the new FX Rates Search widget to search for pairs by currency symbols.

    • Access the new FX Rates Trade Order widget by clicking Place order.

    • Access the new FX Rates Pair Details widget by clicking a currency pair.

  • The new FX Pair Details widget.

    In this widget, the bank customer views up-to-date and detailed information about a selected currency pair. The frequency of automatic data updates is configurable in Experience Manager. The widget provides the following features:

    • View detailed information about a selected currency pair.

    • Access the FX Rates Trade Order widget by clicking Place order.

    • Access the Forex widget by clicking Back to currencies.

  • The new FX Rates Search widget.

    In this widget, the bank customer searches for currency pairs presented in the Forex widget. The widget provides the following features:

    • Search for currency pairs.

    • See the number of matching results.

    • Discard the search query.

  • The new FX Rates Trade Order widget.

    In this widget, the bank customer creates a trade order for a selected currency pair. The widget provides the following main features:

    • Review the exchange rate for the currency pair.

    • Specify the selling and buying amounts.

    • Specify the accounts to withdraw and deposit funds.

  • The new FX Rates Spot Order widget.

    In this widget, the bank customer reviews the created trade order before submitting it. The widget provides the following main features:

    • Review the details of the trade order created in the FX Rates Trade Order widget.

    • Submit or cancel the trade order.

    • Return to the FX Rates Trade Order widget to edit the order.

Known issues

  • Introduced in v3.5.1.

    • MAINT-8565: Account selector accessibility issues.

    • MAINT-8234: Currency inputs accessibility issues.

    • HAD-65: Accessibility screen reader issues on the Place Order modal.

    • HAD-139: When clicking the back button on the Place Order modal, the modal is closed but the cache is not cleared or is not completely cleared.

Payments v3.5.1

This release note explains the changes to this capability.

New features

The new features in this release are:

  • New Stop Сhecks widget

    The new Stop Checks List widget allows the user to view a list of submitted requests to stop the execution of checks.

    The widget has the following features:

    • View a list of submitted stop checks.

    • Click on a specific stop check and view the details.

    • Print the details of a stop check request.

  • Payment editing in the Initiate ACH Credit widget

    While verifying the ACH credit payment details in the Initiate ACH Credit widget, the user can now select the edit option and make the necessary changes to the payment.

  • Improvements in bulk payment approvals

    When a user selects multiple payments and performs an approve or reject action in the Manage Payments widget, the approval action may fail for some of those payments. The user can now view the count of failed approval actions and get detailed information on each of them. Upon clicking the link to view more details, the user can see a list of all payments with failed approvals along with the explanation of the failure.

Bug fixes

The following issues have been fixed in this release:

  • DBSA-5214: The Initiate ACH credit widget has a voice-over accessibility issue in payment reference field.

  • DBSA-5215: The Initiate ACH credit widget has a voice-over accessibility issue in the dropdown TO field.
