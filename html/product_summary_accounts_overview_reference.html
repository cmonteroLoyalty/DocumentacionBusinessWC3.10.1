Selector name

This widget has the following selector name: bb-accounts-overview-widget

Required imports

The following imports are required to set up and customize this widget:

Widget module import

Import the following modules into the module of your wrapper widget:

import { ProductSummaryAccountsOverviewModule } from '@backbase/product-summary-accounts-overview-ang';

    [hasFavoriteAccountsEventName]="hostRef.hasFavoriteAccountsEventName | async"

    [privilege]="hostRef.privilege | async"
    [productKindName]="hostRef.productKindName | async"
    [resourceName]="hostRef.resourceName | async"
    [size]="hostRef.size | async"

    (navigateToTransactions)="hostRef.onNavigateToTransactions($event)"

>
  <bb-accounts-overview-header
    (toggleView)="hostRef.toggleView($event)"
    (sort)="hostRef.sort($event)"

    [isSmallScreen]="hostRef.isSmallScreen"
    [showTitle]="hostRef.hasFavoriteAccounts"
    data-role="accounts-overview-header"

  </bb-accounts-overview-header>

  <ng-container
    *ngIf="aggregatedBalancesPosition === hostRef.aggregatedBalancesPositionEnum.Top && !loadingAggregatedBalance"

    [bbHostRef]="hostRef"
    [bbTemplateContext]="{
      aggregatedBalances: aggregatedBalances,
      aggregatedBalancesPosition: aggregatedBalancesPosition,
      aggregatedBalancesError: aggregatedBalancesError
    }"
  ></ng-container>
  <!--after first time loading, loadingAggregatedBalance will always return false-->
  <bb-product-summary-loading
    [isLoading]="(loading || loadingAggregatedBalance)"
    classes="bb-accounts-overview-loader"
    text="Loading Accounts"
    i18n-text="Loading Accounts@@accounts-overview-container.loading-text"

    <ng-container *ngIf="!error; else errorTemplate">

        </ng-container>
        <div class="bb-card__body">
          <bb-accounts-overview-footer

            (loadMore)="hostRef.onLoadMore()"

          </bb-accounts-overview-footer>
        </div>
      </ng-container>
    </ng-container>
  </bb-product-summary-loading>

    *ngIf="aggregatedBalancesPosition === hostRef.aggregatedBalancesPositionEnum.Bottom && !loadingAggregatedBalance"

      Results found
    </span>
  </div>
</ng-template>

This extension slot consumes the following object properties from its custom context:

Object property Description

accounts: Accounts

Accounts list.

loading: boolean

Defines if data fetching is in progress.

error: AccountsOverviewError

Contains fetching data errors.

paginationParams: GetProductsummaryRequest

Contains pagination parameters.

allowedAccountNavigationTypes: AccountNavigationType[]

Allowed navigations from account, based on user permissions.

isEditAllowed: boolean

Defines if edit actions allowed.

aggregatedBalances: AccountAggregatedBalanceCurrency[]>

List of aggregated account balances per currency.

aggregatedBalancesPosition: AggregatedBalancesPosition

Define position of aggregated balances container.

loadingAggregatedBalance: boolean

Defines the loading state of the balance aggregation.

aggregatedBalancesError: Error

If errors occurred during fetching aggregated balances, holds the errors coming from the server

<ng-template bbAccountsOverviewContainerSwitchViewCustomizable

  let-isSmallScreen="context.isSmallScreen"
  let-allowedAccountNavigationTypes="context.allowedAccountNavigationTypes"

  <ng-container
    [ngSwitch]="activeView">
    <ng-container *ngSwitchCase="'table'">
      <bb-accounts-overview-table

        (navigateToDetails)="hostRef.onNavigateToDetails($event)"

        [isSmallScreen]="isSmallScreen"
        [accounts]="accounts.items"></bb-accounts-overview-table>
    </ng-container>
    <ng-container *ngSwitchCase="'cards'">
      <bb-accounts-overview-cards

    </ng-container>
  </ng-container>
</ng-template>

This extension slot consumes the following object properties from its custom context:

Object property Description

accounts: Accounts

Accounts list.

activeView: string

Active view mode ('table', ‘cards’). Default value is taken from input activeView.

isSmallScreen: boolean

Specifies whether screen size is small. (true if element’s width smaller or equal to 640px).

allowedAccountNavigationTypes: AccountNavigationType[]

Allowed navigations from account, based on user permissions.

isEditAllowed: boolean

Defines if edit actions allowed.

Access Control

A permission is the combination of a business function and a single privilege granted to a user on a given resource type. Privileges are enforced by Digital Banking Services. Widget behavior is affected by the permissions of the user accessing the widget.
