Troubleshooting Angular 10 upgrade

The section provides solutions to common problems when upgrading to Angular 10.

  • Problem: Error when running ng commands: __NG_CLI_RESOURCE__0 is not defined.

    1. Ensure your tsconfig files have the correct module and target settings. Specifically, make sure the module setting is not set to commonjs.

      For the exact settings for each file, see Angular’s migration guide.

  • Problem: npm run package:apps fails because of translation with error ERR! NG_BUILD The translation contains a placeholder with name LINE_BREAK, which does not exist in the message:

    1. Check your translations. Make sure they do not have any HTML such as <br/> inside them.

      For example, change:

      <span i18n="Description@@description">
  Default translation<br/>
  With line break
</span>

      to:

      <div i18n="Description@@description.part-one">
  Default translation
</div>
<div i18n="Description@@description.part-two">
  With line break
</div>

    2. Once the translations are fixed, regenerate them using:

      npm run xi18n

  • Problem: Error in console: Cannot read property 'startSingleApp' of undefined (dev-mock.js).

    1. Extract the page model defined in dev-mock.js file, to the page-model.ts file.

    2. Reference page-model.ts in the environment.ts file.

      For example:

      import { Environment } from './type';
import { ExternalServices } from '@backbase/foundation-ang/start';
import { pageModel } from './page-model';
export const services: ExternalServices = {};

export const environment: Environment = {
  production: false,
  mockProviders: [createMocksInterceptor(), AccessgroupDataHttpMocksProvider, AuditDataHttpMocksProvider],
  bootstrap: {
    pageModel,
    services,
  },
};

    3. Delete the dev-mock.js file, and remove the reference from index.html e.g:

      <script src="./assets/dev-mock.js" type="text/javascript"></script>

      This change was introduced in @bb-cli/schematics version 2.1.0. Read more in the changelog.

  • Problem: Error in console: Cannot set property passive of [object Object] which has only a getter

    1. Remove the following polyfill from your app’s polyfills.ts:

      import 'events-polyfill';
