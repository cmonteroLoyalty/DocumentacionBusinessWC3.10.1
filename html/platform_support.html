Support policies

As a general guideline, Backbase aims to support the latest versions of third-party platforms, frameworks, languages, and libraries, according to the following principles:

  • Upgrade to the latest versions of third-party software as soon as possible. This supports innovation and enables Backbase products to make use of the latest features, security updates, and performance improvements.

  • Maintain a small support footprint by limiting support to the dominant third-party software and accepted industry standards. For example, Backbase only supports the databases and app servers that are used most widely.

  • Maintain consistency in support between Backbase products in cases where it affects customers. This facilitates adoption and helps with interoperability, implementation, and usability.

The following tables outline the support policies for third-party software. These policies apply to all new releases of the Backbase products.

Note
 Backbase products are supported as defined in the End User License Agreement (EULA) with the specified platforms, frameworks, languages, and libraries at the time of release.
