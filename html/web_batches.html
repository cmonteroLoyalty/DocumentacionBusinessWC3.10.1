Web widgets

The web widgets for this capability are:

  • View batch details: View the payments included in a batch, and information about their status.

  • Create manual batch payments: Create a batch manually.

  • Manage batch payments: View a paginated list of batch payments created within the context of the your service agreement.

  • Manage batch file uploads: View a list of uploaded batch files along with the status of upload.

  • Upload batch payment: Upload one or more batch files asynchronously, and track the progress of the upload.

View batch details

This widget provides an overview of the payments included in a batch, and information about their status.

To see the overview of payments for a batch in a paginated list, the user selects the batch from the list in the Manage batch payments widget.

Administrators can configure the widget in Experience Manager to load list items incrementally. To use this feature, the user clicks the Load More button.

To approve or reject a batch, the user clicks Approve or Reject.

If a batch is awaiting approval, the uploader of the batch can delete the batch. To delete a batch, the uploader clicks Delete, and confirms the action in the modal dialog.

Once a batch is approved by at least one approver, any user with the permission to cancel batches can cancel that batch. To cancel a batch, the user clicks Cancel, and confirms the action in the modal dialog.

To go back to the Manage batch payments widget, the user clicks Back to list.

The following details are available for each batch:

  • Batch label

  • Batch type

  • Name of the uploaded file

  • Date and time of the upload

  • Account number, account name, and if present the bank branch code

  • Total credit amount and currency

  • Total debit amount and currency

  • Requested execution date

  • Number of payments

  • Status of the batch

The following details are available for each payment in the batch:

  • Status

  • Counterparty account number, account name, and if present the bank branch code

  • Amount and currency

  • Payment description

If the batch details view is configured with transaction codes turned on, the following details are available for each payment within the batch:

  • Account type:

    Such as Savings • or Checking •, followed by the transaction code.

  • Prenote:

    Displayed under the Payment Description.

  • View details of a batch.

  • View the list of payments included in a batch.

  • View the status of each payment in a batch.

  • Approve or reject the batch.

  • Delete a batch.

  • Cancel a batch.

Create manual batch payments

This widget allows users to manually enter multiple payments through a user-friendly wizard.

The wizard goes through the following steps:

  1. Batch info: Enter the batch name and execution date.

  2. Transfer details: Select the account to transfer from and add payments. For each payment, set the account to transfer to and the amount.

    Optionally, enter a description:

  3. Review: Check the entered payments.

  4. Summary: Shows a success message when the batch order was submitted successfully.

  5. Save as a template: Save a successfully submitted batch as a template for future batch payment submissions.

Create a manual batch from a batch template

The user can create a new batch quickly using a saved template.

  1. Batch info: Select From template to open the list of available templates.

  2. The user can search and select a desired template.

  3. Transfer details: User selects and applies a template. The From template button remains available, the user can use it to select another template from the list. The data will override the current pre-filled fields.

    The template data pre-fills the batch information and transfer details.

  4. Review: Check the entered payments.

  5. Summary: Shows a success message when the pre-filled batch order was submitted successfully.

Features

  • Create a batch order of batch type USA Intracompany

  • Add, edit, and delete payments to the batch order

  • Review before submitting the batch order

  • Create a template from a successfully submitted batch

  • Create a new batch order from a saved template

Services and endpoints

This widget interfaces with the following services:

  • Payments using the payment-batch-client-api

    • The widget calls this endpoint to create the batch order when the user is finished with the Batch info step:

      POST https://ips_host:ips_port/gateway/api/payment-order-service/client-api/v2/batch-orders

    • The widget calls this endpoint to turn dry-run mode on or off and to submit the batch order. To add payments and to submit the batch order, the dry-run mode should be turned off. To submit the batch order for validation, the dry-run mode should be turned on.

      PUT https://ips_host:ips_port/gateway/api/payment-order-service/client-api/v2/batch-orders/{batchOrderID}?dryRun=(false|true)

    • The widget calls these endpoints to create, edit, and delete batch payments in the Transfer details step:

      POST https://ips_host:ips_port/gateway/api/payment-order-service/client-api/v2/batch-orders/{batchOrderID}/batch-payments

      PUT https://ips_host:ips_port/gateway/api/payment-order-service/client-api/v2/batch-orders/{batchOrderID}/batch-payments/{batchPaymentID}

      DELETE https://ips_host:ips_port/gateway/api/payment-order-service/client-api/v2/batch-orders/{batchOrderID}/batch-payments/{batchPaymentID}

    • The widget calls this endpoint to prefetch the batch order list in the Review step of the wizard:

      GET https://ips_host:ips_port/gateway/api/payment-order-service/client-api/v2/batch-orders

  • Products using the arrangement-client-api

    The widget calls this endpoint to fetch arrangements in the Batch info and Transfer details steps of the wizard:

    GET https://ips_host:ips_port/gateway/api/arrangement-manager/client-api/v2/productsummary/context/arrangements

Manage batch payments

This widget provides an overview of batch orders created within the context of the service agreement selected during login.

Batches display in a list, sorted from the most recently uploaded batch order to the oldest, grouped by requested execution date. To sort the list by a different attribute, the user clicks the attribute in the column header. To reverse the sorting order, the user clicks the attribute again.

The user clicks the New batch button to begin the process to upload or create a new batch payment. The options to either create or upload a batch is determined by the user’s account permissions and entitlements. Without the permission Create in one of the batch business functions, the New batch button will not appear.

Pagination is available by default. Administrators can configure the widget in Experience Manager to load list items incrementally. To use this feature, the user clicks the Load More button.

The list of batches can be filtered by the following criteria:

  • To filter by Amount, the user inputs a minimum and a maximum amount in the available fields. The amount filter applies to the total credit amount and the total debit amount of the batch orders. For example, if the amount filter ranges from 200 to 500, the batch orders A, B, and C appear in the list, and batch order D will not:

    • Batch order A: Total credit amount is 300 and total debit amount is 50.

    • Batch order B: Total credit amount is 50 and total debit amount is 300.

    • Batch order C: Total credit amount is 300 and total debit amount is 300.

    • Batch order D: Total credit amount is 50 and total debit amount is 50.

  • To filter by Execution date, the user selects a date range, or a specific date from the calendar.

  • To filter by Status, the user chooses one of the available statuses from the drop-down.

In the search field the user can enter free text for matching on the following criteria:

  • Batch name

  • Own account name

To view details about a batch in the View batch details widget, the user selects the batch from the list.

Credit and debit columns

Whether the Debits and Credits columns are visible, is determined by the value of the creditDebitMixedIndicator key in the response coming from the backend:

  • CREDIT: Shows only the credit column

  • DEBIT: Shows only the debit column

  • MIXED: Shows both columns

The widget has a configuration setting dynamicCreditDebitMixedIndicator that tells the backend how to calculate the creditDebitMixedIndicator key:

  • false (default): Use a fixed configuration setting.

  • true: Use a dynamic calculation. This option results in a slower response time.

If the approval failed for a batch due to limit breach, the approver receives a pop-up notification. To view the detailed breach report, the approver clicks See Details.

Once a batch is approved by at least one approver, any user with the permission to cancel batches can cancel that batch. To cancel a batch, the user clicks Cancel, and confirms the action in the modal dialog.

  • Filter and search the list.

  • Cancel a batch.

Manage batch file uploads

This widget provides the user an overview of their uploaded batch files.

The widget shows uploaded batch files in a list, sorted from the most recently uploaded file to the oldest. The user also sees the status of their upload, that is, if the upload is Done, Open or Failed. To view the error state of a failed upload, the user clicks Details next to the batch file. The error message specifies the following:

  • The number of errors that occurred during upload.

  • The specific error for each line of the file.

If the batch file is a duplicate of another batch file in the same service agreement, it is marked as a DUPLICATE in the list. On selecting the duplicate batch file, the user views its details and any matching previously uploaded file.

The user has three options:

  • Decide later: Closes the dialog, so that the user can return to the decision later.

  • Delete duplicate file: Deletes the recently uploaded file.

  • Keep & Process: The recently uploaded file will be kept for further processing.

Administrators can configure the widget in Experience Manager to load list items incrementally.

To use this feature, the user clicks the Load More button.

Feature

  • View a paginated list of uploaded batch files with their status.

  • View the errors if the upload fails.

Services and endpoints

This widget interfaces with the Payments DBS capability using the payment-batch-client-api.

Upload batch payment

Using this widget, the user selects a batch type and the file or files to upload for that batch type.

The batch payment types available to the user are determined by user permissions.

The upload is asynchronous, that is, the user can navigate away from the widget and perform other actions while the upload is in progress. A progress tracker shows the upload progress per file real time. The user has the option to cancel an upload that is in progress. Once a file upload is completed, the user receives a notification of the final status. If the upload fails, there is a detailed error message available, specifying the following:

  • The number of errors that occurred during upload.

  • The specific error for each line of the file.

When a batch uploads, the widget checks for duplicates by comparing the uploaded batch against other previously uploaded batch files related to the service agreement the current user is operating in. If there is an exact match between the batches parameters, the batch file will be marked as a DUPLICATE. Each character within the file has to be the same in order for it to be marked as a duplicate file.

If a duplicate is found, a sticky notification informs the user that the batch is noted as a DUPLICATE and will not be further processed until further action by the user. The widget displays a modal screen, allowing the user to view the details of the duplicate file and any matching previously uploaded batch.

The user has three options:

  • Decide later: Closes the dialog, so that the user can return to the decision later through the Manage batch file uploads widget.

  • Delete duplicate file: Deletes the recently uploaded file.

  • Keep & Process: The recently uploaded file will be kept for further processing.

To view the uploaded batch payments, the user navigates to the Manage batch file uploads widget directly from the notification.

Features

  • Select a batch file type to upload.

  • Browse for files on your device to upload.

  • Select multiple files to upload at the same time.

  • Upload the selected file or files asynchronously.

  • Track the progress of the upload.

  • Cancel an upload in progress.

  • Receive a notification when the upload is completed.

  • Possible duplicate file detection.

Services and endpoints

This widget interfaces with the following services:

This widget interfaces with the Payments DBS capability using the payment-batch-client-api.
