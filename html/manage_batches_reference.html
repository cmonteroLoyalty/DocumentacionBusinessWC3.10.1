Untitled

Enable dynamic credit/debit/mixed indicator

Determines the value of the dynamicCreditDebitMixedIndicator query parameter when the widget calls the GET batch-orders endpoint. This query parameter instructs the backend how to calculate the creditDebitMixedIndicator key in the response of the same endpoint:

  • false: Use a fixed configuration setting.

  • true: Use a dynamic calculation. This option results in a slower response time.

See also: How to use configure this in DBS.

Access Control

A permission is the combination of a business function and a single privilege granted to a user on a given resource type. Privileges are enforced by Digital Banking Services. Widget behavior is affected by the permissions of the user accessing the widget.

The following table describes how the widget behavior changes depending on the permissions of the user:

Resource

Business function

Privilege

Widget behavior

Batch

Batch - SEPA CT

view

The current user views a list of SEPA credit batch orders.

If the user does not have the permission to view, the widget shows an empty list.

approve
Untitled

The current user is able to approve or reject SEPA credit batch orders.

If the user does not have the permission to approve, the approval and rejection buttons are disabled.

Batch - ACH Credit

view

The current user views a list of NACHA ACH credit batch orders.

If the user does not have the permission to view, the widget shows an empty list.

approve
Untitled

The current user is able to approve or reject NACHA ACH credit batch orders.

If the user does not have the permission to approve, the approval and rejection buttons are disabled.

Batch - ACH Debit

view

The current user views a list of NACHA ACH debit batch orders.

If the user does not have the permission to view, the widget shows an empty list.

approve
Untitled

The current user is able to approve or reject NACHA ACH debit batch orders.

If the user does not have the permission to approve, the approval and rejection buttons are disabled.
