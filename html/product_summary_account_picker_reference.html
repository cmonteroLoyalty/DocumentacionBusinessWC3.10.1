Selector name

This widget has the following selector name: <bb-account-picker-widget></bb-account-picker-widget>

Required imports

The following imports are required to set up and customize this widget:

  • Widget module import

    Import the following modules into the module of your wrapper widget:

    import { ProductSummaryAccountPickerModule } from '@backbase/product-summary-account-picker-ang';

Extension slots

This widget has the following extension slots:

If multiselect is enabled for this widget in Experience Manager.

If multiselect is not enabled for this widget in Experience Manager.

  1. bbAccountPickerWidgetCustomizable

  2. bbAccountPickerContainerCustomizable

  3. bbAccountPickerItemCustomizable

bbAccountPickerWidgetCustomizable

This extension slot includes the following default template:

<ng-template
  bbAccountPickerWidgetCustomizable
  let-hostRef>
    <bb-account-picker-container
      [accountIds]="hostRef.accountIds | async"
      [businessFunction]="hostRef.businessFunction"
      [resourceName]="hostRef.resourceName"
      [privilege]="hostRef.privilege"
      [size]="hostRef.size"
      [multiple]="hostRef.multiple"
      [dropdownPosition]="hostRef.dropdownPosition"
      [favoriteFirst]="hostRef.favoriteFirst"
      [multiSelectLabelLimit]="hostRef.multiSelectLabelLimit"
      [debounceTime]="hostRef.debounceTime"
      [accountsForVirtualScroll]="hostRef.accountsForVirtualScroll"
      [accountsBeforeLoadingMore]="hostRef.accountsBeforeLoadingMore"
      (selectedAccountIds)="hostRef.onSelectedAccount($event)"
    ></bb-account-picker-container>
</ng-template>

This extension slot includes the following default template:

<ng-template
  bbAccountPickerContainerCustomizable

This extension slot consumes the following object properties from its custom context:

Object property Description

accounts

List of accounts to be shown in the selector

multiple

Sets whether the bank customer can select multiple accounts at a time.

Set to false in order to restrict selection to a single account only.

favoriteFirst

Sets how many account names can be displayed at once in the selection field of the widget.

Accounts selected on top of the limit will be truncated.

selectedItems

List of selected items. Type: Observable<ProductsummaryItem | ProductsummaryItem[] | undefined>

loading

Defines if fetching data is in progress. Type: Observable<boolean>

error

Defines error state for the component. Type: Observable<AccountsOverviewError | undefined>

dropdownPosition

Sets the dropdown position. Observable<string>

multiSelectLabelLimit

Sets how many account names can be displayed at once in the selection field of the widget.

Accounts selected on top of the limit will be truncated. Type: Observable<number>

Preferences

Title and description Name Configured by Type Default value

Default business function

In most cases, you should not change the default values.

The widget uses the resource, business function(s) and privilege in the following call to arrangement manager:

GET https://ips_host:ips_port/gateway/api/arrangement-manager/client-api/v2/productsummary/context/arrangements?businessFunction=Product%20Summary,Product%20Summary%20Limited%20View&resourceName=Product%20Summary&privilege=view

The arrangement-manager checks with Access Control whether the user has the privilege for any of the specified business functions. Customers with privileges for the Product Summary Limited View business function only see a limited set of keys for an arrangement, see Limited view.

Access Control

A permission is the combination of a business function and a single privilege granted to a user on a given resource type. Privileges are enforced by Digital Banking Services. Widget behavior is affected by the permissions of the user accessing the widget.

The following table describes how the widget behavior changes depending on the permissions of the user:

Resource

Business function

Privilege

Widget behavior

Product Summary

Product Summary

view

The current user views a list of accounts.

If the user does not have the permission to view any of the two business functions, the widget displays an empty list.

Product Summary Limited View

view

The current user views a list of accounts with limited data. See Limited view.

If the user does not have the permission to view any of the two business functions, the widget displays an empty list.
