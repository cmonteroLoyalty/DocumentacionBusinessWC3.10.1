Selector name

This widget has the following selector name: <bb-notifications-badge-widget>.

Required imports

The following imports are required to set up and customize this widget:

  • Angular Common module import:

    import { CommonModule } from '@angular/common';

  • Backbase Core module import:

    import { BackbaseCoreModule } from '@backbase/foundation-ang/core';

  • Widget module import:

    import { NotificationsBadgeWidgetModule } from '@backbase/notifications-badge-widget-ang';

Extension slots

This widget has the following extension slots:

  1. bbNotificationsBadgeCustomizable

  2. bbNotificationsListCustomizable

  3. bbNotificationsListItemCustomizable

  4. bbNotificationDetailsCustomizable

  5. bbNotificationDetailsItemCustomizable

  6. bbNotificationsCapabilityIconCustomizable

  7. bbHumanizedDateCustomizable

bbNotificationsBadgeCustomizable

This extension slot includes the following default template:

<ng-template
  bbNotificationsBadgeCustomizable
  let-hostRef
  let-unreadCount="context.unreadCount"
  let-listTransform="context.listTransform"
>
  <div class="bb-notifications-badge-widget">
    <div #popover="ngbDropdown" ngbDropdown placement="bottom-right" (openChange)="hostRef.onOpenChange($event)">
      <button
        bbButton
        [attr.aria-labelledby]="'notifications-badge-label' + hostRef.id"
        color="transparent"
        class="bb-notifications-badge-widget__dropdown-button"
        ngbDropdownToggle
      >
        <bb-icon-ui name="notifications" color="secondary" data-role="notifications-icon" size="lg"></bb-icon-ui>
        <bb-badge-counter-ui
          class="bb-notifications-badge-widget__dropdown-button-counter"
          [count]="unreadCount"
          maxCount="99"
          color="danger"
          data-role="notifications-unread-count"
        ></bb-badge-counter-ui>
      </button>
      <div
        ngbDropdownMenu
        class="bb-notifications-badge-widget__dropdown-menu dropdown-menu-unstyled"
        data-role="notifications-list-wrapper"
      >
        <div
          class="bb-notifications-badge-widget__dropdown-menu-list"
          data-role="notifications-dropdown-menu-list"
          [style.transform]="listTransform"
        >
          <div class="card bb-notifications-badge-widget__list-wrapper">
            <bb-notifications-list
              *ngIf="popover.isOpen()"
              (navigation)="popover.close()"
              [preselectedNotification]="hostRef.preselectedNotification$ | async"
            ></bb-notifications-list>
          </div>
        </div>
      </div>
      <span
        i18n="Notifications badge aria label@notifications.badge.aria-label"
        [attr.id]="'notifications-badge-label' + hostRef.id"
        aria-live="polite"
        aria-relevant="text"
        class="sr-only"
      >
        { unreadCount || 0, plural, =0 { no new notifications } =1 { {{ unreadCount }} new notification} other {
        {{ unreadCount }} new notifications } }
      </span>
    </div>
  </div>
</ng-template>

          {{ notifications.length }} new notifications of {{ totalCount }}available } }

            </bb-notifications-list-item>

        </ul>

            <div

                i18n="Load more button@@notifications.notification-list.load-more"

                You’ve reached the end of the list.

      ></bb-notification-details>
    </ng-container>
  </ng-container>

  <div role="status">
    <div class="card-body" *ngIf="!notifications?.length || error">
      <bb-empty-state-ui

        data-role="notifications-badge-error-state"

bbHumanizedDateCustomizable

This extension slot includes the following default template:

<ng-template bbHumanizedDateCustomizable let-hostRef>
  <div class="sr-only" i18n="Humanized date title@@humanized.date.title">,Date:</div>
  <ng-container *ngIf="hostRef.date as date">
    <time *ngIf="hostRef.isToday" class="bb-subheader bb-subheader--regular bb-text-nowrap">
      <ng-container *ngIf="hostRef.displayRelativeTime; else today">
        <ng-container
          *ngIf="hostRef.isWithinHour; else moreThanHourAgo"
          i18n="Notifications time less than an hour ago date@@notifications.humanized-date.less-than-an-hour-ago"
        >
          { hostRef.minutesFromNow, plural, =0 {now} other {{{hostRef.minutesFromNow}} min ago} }
        </ng-container>

        <ng-template #moreThanHourAgo>
          {{ date | bbDate: 'shortTime' }}
        </ng-template>
      </ng-container>

      <ng-template #today i18n="Notifications today date@@notifications.humanized-date.today"> today </ng-template>
    </time>

    <time
      *ngIf="hostRef.isYesterday"
      class="bb-subheader bb-subheader--regular"
      i18n="Notifications yesterday date@@notifications.humanized-date.yesterday"
    >
      yesterday
    </time>

    <time
      *ngIf="!hostRef.isToday && !hostRef.isYesterday"
      class="bb-subheader bb-subheader--regular"
      [attr.aria-label]="date | bbDate: 'mediumDate'"
      role="text"
    >
      {{ date | bbDate: 'MM/dd/yy' }}
    </time>
  </ng-container>
</ng-template>

API

The following table shows the public class members of the host component that you access from each extension slot:

Component Extension slot identifier Public class member Description

HumanizedDateComponent

bbHumanizedDateCustomizable

@Input() displayRelativeTime: boolean

Flag that defines if we should display relative time for today.

@Input() set date(value: Date | string | undefined)

Notification date.

get date(): Date | string | undefined

Date getter.

get isWithinHour(): boolean

Is date within an hour from now flag.

get isToday(): boolean

Is date within the current day flag.

get isYesterday(): boolean

Is date within the yesterday date flag.

get minutesFromNow(): number | undefined

How many minutes from now is the date getter.

NotificationDetailsItemComponent

bbNotificationDetailsItemCustomizable

readonly levelMap: Readonly<SeverityLevelIconMap>

The notifications severity map, which includes the severity level icon and the color mapper.

@Input() notification: NotificationItem

The notification item data.

readonly displayRelativeTime$: Observable<boolean>

The stream of the widget preference with a flag that defines if relative time for today should be displayed.

get icon(): string

The getter for the icon name from the severity level icon map depending on the severity level of the notification.

get color(): string

The getter for icon color from the severity level icon map depending on the severity level of the notification.

NotificationDetailsComponent

bbNotificationDetailsCustomizable

@Input() notification: NotificationItem

The notification item data.

@Output() navigateBack: EventEmitter<void>

The event to navigate back to the notification list.

@Output() delete: EventEmitter<void>

Event to delete notification.

deleteNotification(): void

Method to emit delete notification event.

NotificationsListComponent

bbNotificationsListCustomizable

@Output() navigation: EventEmitter<void>

Navigation event.

@Input() set preselectedNotification(notification: NotificationItem | undefined)

Preselected notification.

readonly allowedRoutes$: Observable<string[]>

Routes available for navigation.

readonly displaySettingsButton$: Observable<boolean>;

The stream of the flag that defines whether the settings button should be displayed.

readonly notifications$: Observable<Notifications>

A stream with notification items.

readonly error$: Observable<Error>

Notification loading request error stream.

readonly markAllAsReadError$: Observable<boolean>

Mark-as-read request error stream that shows if the request has failed.

readonly isLoading$: Observable<boolean>

A stream with a flag that shows if notifications are currently loading.

Display relative time for today

Whether the exact date of receiving a notification should be changed to a more convenient relative format. If this preference is set to true, then the following rules apply:

  • If the receipt time is less than 1 minute ago, the timestamp changes to Now.

  • If the receipt time is more than 1 minute but less than 1 hour ago, then the number of minutes is displayed. For example, 59 min ago.

  • If the receipt time is more than 1 hour but less than 1 day ago, then the HH:MM AM/PM format is used. For example, 10:00 AM.

  • If the receipt time is 1 day ago, the timestamp changes to Yesterday.

  • If the receipt time is more than 1 day ago, then the DD/MM/YYYY format is used. For example, 10/03/2020.
