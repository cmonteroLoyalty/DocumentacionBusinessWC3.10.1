Notifications v3.9.2

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Accessibility improvement with the Load more button

    When the bank customer selects to load more notifications in the Notification badge widget, the focus is moved to the first new notification. In that way, keyboard users will not miss out on any information and will be able to go through the notifications in the same logical order as other users.

  • Web: Accessibility improvement with read/unread status of the notification

    The hidden text was added to the title of the notification in the Notification badge widget. The title indicates to bank customers whether the notification is read or unread.

Payments v3.9.2

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Create payment template

    The Initiate omni payment widget now supports creating payment templates without creating a payment. To enable this features, set the isTemplateMode to true in the Initiate omni payment widget’s options in the wrapper widget, see Configure create payment template

    The template form is a modal window where users select an available payment type. The widget validates the specified payment details before storing the payment template.

  • Web: Enter purpose of payment for international payments

    The Initiate omni payment and Initiate payment wizard widgets now support entering a valid purpose of payment for international payments. The purpose of payment is a code or a free text field, which is configurable per country on the backend. The widget handles this as follows:

    • If the configured format is a free text, the user is able to enter a text for the purpose of payment up to 35 characters. The free text format is mandatory or optional.

    • If the configured format is a code, the user selects a purpose of payment code from a predefined list. The predefined list shows the purpose of payment codes and their descriptions. The code format is always mandatory.

  • Web: Show purpose of payment in details of a payment

    The Initiate omni payment and Initiate payment wizard widgets now support showing the purpose of payment in the payment details for international payments, if it is provided with the payment. The purpose of payment is a code or a free text field, which is configurable per country on the backend. The widget handles both formats and for the code format, also shows the associated description.

  • Web: Show validation error messages

    The Initiate omni payment and Initiate payment wizard widgets now show the validation error messages coming from the backend after validating the entered payment details and after an attempt to submit the payment.

  • Web: Filter default country from the dropdown for international payments

    The Initiate omni payment widget now filters and lists countries marked as default in the target country dropdown for international payments.

Bug fixes

This release includes the following bug fixes:

  • MAINT-11293: Omni Payment Widget: selectedContact should be pre-filled when a contact is selected

    In the Initiate omni payment and Initiate payment wizard widgets, while initiating a payment using an existing contact, the selected contact’s metadata, such as the contact ID, was not sent to DBS. This has been fixed. The selected contact metadata are included as part of the initiate payment payload.

Deprecations

This release includes the following deprecation:

  • In v3.9.2:

    • Refactoring of the manage-payments container means that some components are removed. The ManagePaymentsContainerComponent`s listed below use dialogs for `[isOpen]/(cancel) prior this release:

      • showApproveConfirmationDialog and cancelConfirmationActionApprove

      • showRejectConfirmationDialog and cancelConfirmationActionReject

      • showCancelConfirmationDialog and cancelConfirmationActionCancel

      • showBulkApproveConfirmationDialog and cancelConfirmationActionBulkApprove

      • showBulkRejectConfirmationDialog and cancelConfirmationActionBulkReject

      In this release, these dialogs are removed and the following dialogs are introduced:

      • (cancel)="hostRef.cancelConfirmationDialog()": Close any type of confirmation dialog

      • [isOpen]="hostRef.isConfirmationDialogOpen(type)": Open a specific confirmation dialog

Products v3.9.2

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Select business account widget has an accessibility improvement

    The Select business account widget now conforms to WCAG 2.1-AA, except for one third-party component: ng-select.

Known issues

This release has the following known issue:

  • Discovered in v3.9.2

    • The Select business account web widget uses a third-party component, ng-select, that doesn’t conform to the WCAG 2.1-AA accessibility standard.

Transactions v3.9.2

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: View enlarged check images, print, and open in a separate browser tab

    Bank customers can now view check images enlarged in a lightbox component. From the lightbox component, the following features are available:

    • Print check images.

    • Open check image in a separate browser tab.

    • Zoom in and out with button controls.

    • Download check images.
