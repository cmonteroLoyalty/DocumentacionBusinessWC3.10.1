All capabilities v3.5.3

This release note explains changes that affect multiple capabilities in this release.

Deprecations

This release includes the following deprecations:

  • In v3.6.0:

    • Artifacts demo-ang-<version>-corporate.zip and demo-ang-<version>-corporate-portal.zip will be renamed to demo-ang-<version>-business.zip and demo-ang-<version>-business-portal.zip respectively.

Authentication v3.5.3.1

This release note explains changes to this capability.

New features

  • Improved widget title

    To make its purpose clearer and improve consistency among widget collections, the Login widget has been retitled to Log in user widget.

Bug fixes

The following issues have been fixed in this patch:

  • MAINT-9389: Unable to use custom provider tokens for the Log in user widget

    To resolve this, providers are now added at the module level instead of the component level.

  • MAINT-9415: Users could only log in on the second attempt

    To resolve this, the Log in user widget has been updated so that the login post request now includes the responseType set to text.

To update to v3.5.3-sp.1, run the following command from your project application directory:

npm i --save @backbase/retail-ang@3.5.3-sp.1

Approvals v3.5.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Mobile: Approvals journey

    The Approvals journey shows a paginated list of approvable payment orders for the current context.

Authentication v3.5.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Config providers, services, and components are now exported from the public API

    Configuration providers, services, and components are now exported from the Login widget public API. Including these elements in the API ensures that they are publicly accessible.

Batches v3.5.3

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • CB-4514: Values appear with a lag when typing in the field to filter batches by amount.

  • CB-4556: Batch duplicate notification is read twice (accessibility).

  • CB-4574: The batch status filter does not show the CANCELLED status.

Contacts v3.5.3.2

This release note explains changes to this capability.

Bug fixes

This patch includes the following bug fixes:

  • MAINT-9891: Access to the whole error response

    There was an inconsistency in the error schema used by Contacts RAML specs. They were using a custom schema different from the Service SDK one. The schema for generic errors has been updated to use the same one as defined in Service SDK. The Contact Create and Edit business contact widgets now use the correct schema and expose it to the corresponding templates.

Contacts v3.5.3.7

This release note explains changes to this capability.

Bug fixes

This patch includes the following bug fixes:

  • MAINT-10110: Unable to customize notification messages in contact manager

    Notification messages for Contact Create, Contact Details and Contact Edit widgets were not customizable.

  • MAINT-10319: Notifications are not visible when extending Contact Manager My Requests and Contact Manager Pending Approvals

    When customizing extension slots for Contact Manager My Requests and Contact Manager Pending Approvals widgets, notification messages were set to blank.

Deprecations

Payments v3.5.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Omni-payment widget (web)

    The Omni-payment widget is a base widget that is fully customizable with a TypeScript configuration. It comes with the following out-of-the-box configuration:

    • Create wire (US): Configures the widget for outgoing US domestic wire transfers.

    The Omni-payment widget always requires a developer to either configure an out-of-the-box payment type, or to create and configure a custom payment type. For more information, see:

    • Configure an out-of-the-box payment type (beta)

    • Configure a custom payment type (beta)

    Note
    		 This is a beta (sample code) release. This feature may, unless otherwise authorized by Backbase, be used for testing purposes only, and is provided on an "AS IS" basis without support of any kind. You should not use this feature in a production environment.

Migration

There are no migration steps specific to this release.
