All capabilities v3.6.0

This release note explains changes that affect multiple capabilities in this release.

Breaking changes

This release includes the following breaking changes:

  • Artifacts demo-ang-<version>-corporate.zip and demo-ang-<version>-corporate-portal.zip have been renamed to demo-ang-<version>-business.zip and demo-ang-<version>-business-portal.zip respectively.

  • Microsoft Internet Explorer 11 is deprecated.

  • Microsoft Internet Explorer 10 is removed.

  • Only all triangular capabilities or all lean capabilities together with triangular ones should be used.

    The lean capabilities in Digital Banking Services are:

    • Contacts

    • Messages

    • Notifications

    • Payments

    The lean capability in Entitlements Services is:

  • Approvals

Account statements v3.6.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • View and download business account statements widget

    A new widget has been added to the Widget Collection Business Banking. The widget allows the bank customer to:

    • View a list of available account statements for the selected account.

    • Sort and filter the list by date range.

    • Download account statements in different formats.

    • View account statements in the PDF format.

    Read more in Understand View and download business account statements.

Actions v3.6.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Configure business approval notification settings widget

    A new widget has been added to the Widget Collection Business Banking. The widget allows the bank customer to set up notifications for the following events:

    • When a payment created by them is rejected.

    • When a contact change submitted by them is rejected.

    Read more in Understand Configure business approval notification settings.

  • New entry point to notification settings:

    A new entry point to notification settings has been introduced for business banking via the User Profile page.

Payments v3.6.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Add a comment while approving or rejecting a payment order

    While approving or rejecting a single payment order from the Manage Payments widget, a user can now add a comment to complement his action. This can be used to specify a reason why the payment is rejected, or provide a nice remark for the subsequent approver(s).

  • View the approval comments in the approval log for a selected payment order

    When viewing the details of a payment from within the Manage Payments widget, a user can now view the comments, if any, added by the approver(s) in the approval log.

  • Widgets have been migrated to Theme 2.0

    The following widgets have been migrated to the new theme:

    • Initiate Payment

    • Initiate Business Payment

    • ACH Credit Transfer

    • Manage Payments

    • Request Stop Checks

    • Stop Checks List

  • Widgets assigned new titles

    The following web widgets were assigned new titles in order to have consistent and synchronised naming across applications.

    • The Stop Checks List widget title changed to View stop check payment requests.

    • The Omni payment widget title changed to Initiate omni payment.

  • Initiate omni payment.

    The Initiate omni payment widget is a base widget that is fully customizable with a TypeScript configuration. The widget always requires a developer to either configure an out-of-the-box payment type, or to create and configure a custom payment type.

    This widget ships with out-of-the-box configurations for the following payment types:

    • Domestic wire (US)

    • ACH credit transfer (US)

    For more information, see Adopt the Initiate omni payment widget

Bug fixes

This release includes the following bug fixes:

  • MAINT-9275: Initiate Payments (web) - error from the /validate endpoint is ignored

    When the widget sent a request to the endpoint POST //payment-order-presentation-spec/client-api/v2/payment-orders/validate to validate the payment payload, the error, if any received, was ignored. This has been fixed, and the user flow is now stopped if validation fails.

Deprecations

This release includes the following deprecations:

  • The Initiate ACH Credit widget is deprecated and will be removed in v3.7.0.

    This is now replaced with the Initiate omni payment widget with out-of-the-box configuration for ACH credit transfer.

    • In v3.7.0:

  • Support ceases for the Initiate ACH credit widget.

Breaking changes

This release includes the following breaking changes:

  • The following deprecated properties have been removed from the Initiate Payment, Initiate Business Payment, and Manage Payments widgets:

    Removed

    Replacement

    debtor/debtorAccount

    originator/originatorAccount

    creditTransferTransactionInformation

    transferTransactionInformation

    creditTransferTransactionInformation.creditor

    transferTransactionInformation.counterparty

    creditTransferTransactionInformation.creditorAccount

    transferTransactionInformation.counterpartyAccount

    creditTransferTransactionInformation.creditorBank

    transferTransactionInformation.counterpartyBank

Known issues

This release has the following known issues:

  • Discovered in v3.6.0

    • MAINT-9720: Cannot have two omni-payment widgets with different payment types.

      When two or more widgets customize the Initiate omni-payment widget and use the withConfig method to provide the payment configuration, the second or last configuration made overrides all the previous configurations.

      To work around this issue, provide the paymentConfig and the business function as inputs to the Initiate omni-payment widget component, in the wrapper widget’s template. Do not use the withConfig method. This issue is fixed with the 3.6.2 release.
