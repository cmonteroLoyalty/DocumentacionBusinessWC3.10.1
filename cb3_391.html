Business Banking Widget Collection v3.9.1

The highlights for this release are:

  • Capabilities

    • Notifications v3.9.1

      • Delete notification

    • Payments v3.9.1

      • Show currency rate and converted amount after applying a cross-currency template

      • Backdating is disabled for new payment orders

    • Products v3.9.1

      • View business account list widget has an accessibility improvement.

      • View business account balance history widget has an accessibility improvement

      • Bug fixes

    • User Self Service v3.9.1-sp.2

      • Bug fix

  • Migration

To find out which Backbase product versions have been verified together, see Backbase product compatibility.

Notifications v3.9.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Delete notification

    The delete notification functionality has been added to the Notification badge widget. This allows the customer to remove obsolete notifications from the list.

Payments v3.9.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Show currency rate and converted amount after applying a cross-currency template

    Applying a cross-currency template for a payment also shows currency rates and the converted amount to the customer. Both the Initiate omni payment and the Initiate payment wizard widgets have this feature.

  • Web: Backdating is disabled for new payment orders

    The date selector no longer allows backdating in the following situations:

    • One-off payment order: When entering the execution date, selection of any date prior to the current date is disabled.

    • Standing payment order: When entering the start date, selection of any date prior to the current date is disabled.

    • Standing payment order: When entering the end date, selection of any date prior to the minimum end date, which is calculated based on the selected frequency, is disabled.

    Both the Initiate omni payment and the Initiate payment wizard widgets have this feature.

Deprecations

This release includes the following deprecations:

  • In v3.10.0:

    • Support ceases for the bbManagePaymentsContainerCustomizableDirective extension slot in the Manage payments widget. In stead, use one of the following extension slots:

      • bbManagePaymentsListCustomizableDirective

      • bbManagePaymentsDetailsCustomizableDirective

      • bbManagePaymentsDetailsListCustomizableDirective

      • bbLoadingCustomizableDirective

      • bbEmptyListCustomizableDirective

      • bbManagePaymentsListHeaderCustomizableDirective

Products v3.9.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: View business account list widget has an accessibility improvement

    The View business account list widget now conforms to WCAG 2 level AA. This is applicable for users using a keyboard or a screen readers.

  • Web: View business account balance history widget has an accessibility improvement

    The View business account balance history widget now conforms to WCAG 2 level AA. This is applicable for users using a keyboard or a screen readers.

Bug fixes

This release includes the following bug fixes:

  • MAINT-11129: Missing pagination parameters in bb-product-summary-footer-ui

    In the View business account list widget, it was not possible to set up direction links for pagination through the bbAccountsOverviewFooterCustomizable extension slot by using the bb-product-summary-footer-ui.

  • MAINT-11194: Inconsistent and wrong dates in the balance history graph

    Dates in the balance history graph were shown in various formats including: "Dec 13" and "fri 25", "November" rather than "Nov". The bug is fixed and the dates are now formatted in a uniform manner.

User Self Service v3.9.1-sp.2

This release note explains changes to this capability.

Bug fixes

This patch includes the following bug fix:

  • IDLM-166: Support for additions

    Following the migration to OpenAPI, a number of request and response objects did not support additional properties (additions).

    All objects in the user-http-ang 3.0.1-sp.1 data module have now been updated to support additions.

Migration

When you migrate from one version of this product to another, best practice is to update incrementally through all minor and major versions in between.

Follow these steps to update an existing deployment:

  1. Extension slot for Manage payments widget deprecated

    If you have an extension for the bbManagePaymentsContainerCustomizableDirective extension slot in the Manage payments widget, replace it with one of the following extension slots:

    • bbManagePaymentsListCustomizableDirective

    • bbManagePaymentsDetailsCustomizableDirective

    • bbManagePaymentsDetailsListCustomizableDirective

    • bbLoadingCustomizableDirective

    • bbEmptyListCustomizableDirective

    • bbManagePaymentsListHeaderCustomizableDirective
