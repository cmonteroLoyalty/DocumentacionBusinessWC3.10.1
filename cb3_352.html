All capabilities v3.5.2

This release note explains changes that affect multiple capabilities in this release.

New features

The new features in this release are:

  • Journeys v1.0.0 beta (sample code)

    This release introduces the journey approach for native mobile. Journeys are a set of screens with interlinked navigation. Each screen is a page, hosted in a capability.

    Journeys cover one or more capabilities and offer projects a well tested user experience. They are configurable and themeable for each project, using the new Mobile Design System.

    Widget Collection Business Banking includes the following out-of-the-box journeys:

    • Authentication journey

      The Authentication journey includes the following features from the Authentication capability and the Access Control capability:

      • Enroll with a username and password.

      • Log in with a username and password.

      • Set up biometrics and use them to log in.

      • Select or change the service agreement.

      The journey implementor can configure the Authentication journey to meet their project’s needs.

    • Accounts and Transactions journey

      The Accounts and Transactions journey includes following features from the Products capability and the Transactions capability:

      • Show the list of accounts associated with the selected service agreement.

      • Search within the accounts list associated with the selected service agreement.

      • View the complete details of the selected account.

      • Show the list of transactions for an account.

      • Show the Trend-Graph for a 30-day period (from the current date).

      • Show details for a selected transaction.

      The journey implementor can configure the Accounts and Transactions journey, the Accounts List, and the Transactions List.

  • Business Banking App beta (sample code)

    The Business Banking App for native mobile offers a configurable, out-of-the-box implementation of journeys and capabilities.

    The Business Banking App also includes the More Menu. This menu provides the following features:

    • Show a menu list that includes the current logged-in user.

    • Change the service agreement (if there is more than one) without restarting the app.

    • Log out from the app and clean all user information.

Note
 These are beta (sample code) features. They are unsupported and subject to breaking changes.

Batches v3.5.2

This release note explains the changes to this capability.

New features

The new features in this release are:

  • Notify the user of a limit breach during a batch approval

    When a user approves a batch payment that would result in the approval limit being exceeded, the user receives a notification about the limit breach. This notification provides the following details:

    • Batch information, including the name, number of payments, batch amount, and currency.

    • Breach limit information, including the type of limit breached (transaction or period), set limit, and consumed limit.

    Batch approvals are submitted using the following endpoint:

    PUT https://ips_host:ips_port/gateway/api/payment-order-presentation-service/client-api/v2/batch-orders/{batchOrderId}/approvals

  • Filter batch payments

    The Manage Batches web widget now provides the ability to filter batch payments by the following criteria:

    • Payment amount: The user can specify a minimum and a maximum amount.

    • Execution date: The user can specify a specific date or date range.

    • Status: The user can select one or more statuses.

    Batch payments are filtered using the following endpoint:

    GET https://ips_host:ips_port/gateway/api/payment-order-presentation-service/client-api/v2/batch-orders/{batchOrderId}

  • Cancel batch payments

    If at least one approver has approved a batch payment and the batch status is still ENTERED, any user with the correct permissions can cancel that batch. To cancel a batch in the Manage Batches web widget, the user clicks Cancel in the batch details and confirms the action in the modal dialog.

    Batch payments are cancelled using the following endpoint:

    PUT https://ips_host:ips_port/gateway/api/payment-order-presentation-service/client-api/v2/batch-orders/{batchOrderId}

    Bug fixes

    The following issues have been fixed in this release:

  • CB-4172 accessibility issue: The screen reader did not read out the modal information after the user selected a file to upload.

  • CB-4176 accessibility issue: When the screen displayed Batch file is a duplicate notification, the screen reader read out items from the previous modal dialog.

Known issues

The known issues in this release are:

  • Introduced in v3.5.2

    • CB-4514: Values appear with a lag when typing in the field to filter batches by amount.

    • CB-4556: Batch duplicate notification is read twice (accessibility).

    • CB-4574: The batch status filter does not show the CANCELLED status.

Payments v3.5.2

This release note explains the changes to this capability.

New features

The new features in this release are:

  • Updated the Collect from drop-down list for ACH Debit

    To improve consistency, the drop-down list in the Collect from section of the ACH Create Debit template has been updated to match the drop-down lists used for other types of payments.

  • Submit a request to stop an issued check

    The Stop Checks List widget enables a bank customer to submit a request to the bank to stop the clearance of a check (or cheque). This feature is helpful in the following scenarios:

    • The customer has insufficient funds and wants to prevent the check from bouncing.

    • The customer entered the incorrect amount or other details on the check.

    • The customer’s check or check book was lost or stolen. In the case of multiple checks or an entire check book, the customer can stop all checks that fall within a range of serial numbers.

    • The customer mailed the check to the wrong address or it was lost in the mail.

    • The customer has a dispute over purchases.

Bug fixes

The following issues have been fixed in this release:

  • NEOP-337: Accessibility issues in Create Debit - the user was not able to navigate using tabs in the Contacts section, after selecting the Contacts icon.

  • NEOP-639: Accessibility issues with opening drop-down lists in the Manage Debits widget - the More drop-down menu was not expanded by pressing the Space or Enter key.

  • NEOP-472: Accessibility issues with selecting options in drop-down lists in the Manage Debits widget - The More drop down menu items like Copy and Cancel were not accessible via keyboard arrow keys.

  • NEOP-641: Accessibility issues in Create Debits - bank customers could not access input fields for Client address and Bank address sections using the keyboard tab navigation option.

  • NEOP-642: Accessibility issues in Create Debits - bank customers could not access the buttons Edit, Discard, and Submit using the tab keyboard navigation option.

Products v3.5.2

This release note explains the changes to this capability.

New features

The new features in this release are:

  • Migration of the Favorite Accounts widget to WC3

    The Favorite Accounts widget is migrated to WC3. This widget enables bank customers to:

    • View their favourite accounts in card view.

    • Add accounts to the Favourite Accounts list.

    • Remove accounts from the Favourite Accounts list.

    To add an account to the favorites, the customer clicks the star icon associated with that account in the Product Summary Accounts Overview widget. It is possible to add accounts to the favorites from both list and card views.

    The horizontal list of favorite accounts is scrollable until the last favorite account is displayed.

  • Configure chart legend visibility in the Product Summary Balance History widget

    It is now possible to configure the Product Summary Balance History widget in Experience Manager in order to show or hide the chart legend.

  • New Products page beta (sample code)

    The new Products page for native mobile provides the following features:

    • Show the list of accounts associated with the selected service agreement.

    • Search within the accounts list associated with the selected service agreement.

    • Configure the accounts list.

    • View the complete details of the selected account.

    • Configure the account details.

Bug fixes

The following issues have been fixed in this release:

  • PRSUM-4928: Accessibility issue on the Balance History widget - the NVDA reader should announce “Download options menu button collapsed sub menu” when the user tabs to the download icon and the menu is collapsed.

  • PRSUM-5162: Accessibility issue in the cards view of Accounts Overview - bank customers could not sort accounts with the drop-down sorting fields using the ENTER keyboard navigation option.

Migration

When you migrate from one version of this product to another, best practice is to update incrementally through all minor and major versions in between.

Follow these steps to update an existing deployment:

  1. Migration of Messages widgets to the Communication API

    To enable Messages widgets to use the new Communication API, add a feature flag in your app.module:

    BackbaseCoreModule.forRoot({
  logDeprecations: true,
  features: {
    NEW_COMMUNICATION_API_ENABLED: true,
  },
})

    Find more information about this configuration in Communication between widgets.
