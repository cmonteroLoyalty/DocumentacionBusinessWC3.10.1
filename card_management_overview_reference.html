View cards

This page includes the following information:

  • Selector name

  • Required imports

  • Extension slots

  • Implementation example

  • Preferences

Selector name

This widget has the following selector name: <bb-cards-list-widget>.

Required imports

The following imports are required to set up and customize this widget:

  • Widget module import

    Import the following modules into the module of your wrapper widget:

    import { CardsListWidgetModule } from '@backbase/retail-ang/cards';

  • Widget component import

    Import the following components into the module of your wrapper widget:

    import { CardsListWidgetComponent } from '@backbase/retail-ang/cards';

  • Data module imports

    Import the following data modules into the module of your wrapper widget:

    import { CardsDataHttpModule } from '@backbase/data-ang/cards';

Extension slots

This widget has the following extension slots:

  1. bbCardsListPaymentCardControlsCustomizable

  2. bbCardsListPaymentCardCustomizable

  3. bbCardsListPaymentCardsContainerCustomizable

  4. bbCardsListPaymentCardsEmptyStateContainerCustomizable

  5. bbCardsMultipleErrorStateCustomizable

You configure these extension slots using:

bbCardsListPaymentCardControlsCustomizable

This extension slot includes the following default template:

<ng-template bbCardsListPaymentCardControlsCustomizable let-hostRef>
  <ng-container *ngIf="hostRef.paymentCard.status === 'ACTIVE'">
    <span class="bb-payment-card-controls">
      <bb-load-button-ui
        color="secondary"
        [isLoading]="hostRef.isLockLoading"
        (click)="!hostRef.isLockLoading && hostRef.onLock()"
        i18n="Lock button|Control card by locking@@card.control.lock"
        *ngIf="!hostRef.isLocked"
      >Lock</bb-load-button-ui>
      <bb-load-button-ui
        color="primary"
        [isLoading]="hostRef.isLockLoading"
        (click)="!hostRef.isLockLoading && hostRef.onUnlock()"
        i18n="Unlock button|Control card by unlocking@@card.control.unlock"
        *ngIf="hostRef.isLocked"
      >Unlock</bb-load-button-ui>
    </span>

    <bb-payment-card-initiate-replacement
      [paymentCard]="hostRef.paymentCard"
      [isInitiateReplacementLoading]="hostRef.isInitiateReplacementLoading"
      (initiateReplacement)="hostRef.onInitiateReplacement($event)"
    ></bb-payment-card-initiate-replacement>
  </ng-container>

  <ng-container *ngIf="hostRef.paymentCard.status === 'INACTIVE'">
    <bb-payment-card-initiate-activation
      [paymentCard]="hostRef.paymentCard"
      [isInitiateActivationLoading]="hostRef.isInitiateActivationLoading"
      (initiateActivation)="hostRef.onInitiateActivation($event)"
    ></bb-payment-card-initiate-activation>
  </ng-container>
</ng-template>

bbCardsListPaymentCardsContainerCustomizable

This extension slot includes the following default template:

<ng-template bbCardsListPaymentCardsContainerCustomizable let-hostRef>
  <ng-container *ngIf="hostRef.paymentCardGroups.length">
    <ng-container *ngFor="let paymentCardGroup of hostRef.paymentCardGroups">
      <bb-payment-card-group
        *ngIf="paymentCardGroup.paymentCards.length"
        class="bb-block bb-block--sm"
        [heading]="paymentCardGroup.type"
        [paymentCards]="paymentCardGroup.paymentCards"
        [notificationTtl]="hostRef.notificationTtl"
        (selectCard)="hostRef.onSelectCard($event)"
      ></bb-payment-card-group>
    </ng-container>
  </ng-container>
  <ng-container *ngIf="!hostRef.paymentCardGroups.length">
    <bb-payment-card-group
      [paymentCards]="hostRef.paymentCards"
      [notificationTtl]="hostRef.notificationTtl"
      (selectCard)="hostRef.onSelectCard($event)"
    ></bb-payment-card-group>
  </ng-container>
</ng-template>

Preferences

This widget has the following preferences that managers or administrators configure in Experience Manager:

Title and description Name Admin[a] Default value
Untitled

Group by Payment Card Types

Configure the card types that the widget displays:

  • 'Credit,Debit' - display credit cards and debit cards. Cards are grouped per type. A label indicates the card type.

  • 'Credit' - display credit cards with label.

  • 'Debit' - display debit cards with label.

  • Empty - display all cards with no label.

The value is a comma-separated string. Each value corresponds to the type as returned by the backend service.
