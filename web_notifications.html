Web widgets

This capability includes the following widgets:

  • Receive notifications

    • Notification badge: View the number of unread notifications.

    • View sticky and pop-up notifications: View and dismiss pop-up and banner notifications.

  • Manage notification preferences

    • Manage business approval notification preferences: Set up notification preferences.

Notification badge

The Notification badge widget alerts any bank customer of unread notifications in their inbox. The recipient does not need any specific business function privileges from Access Control to read and manage notifications.

A notification count is displayed when the bank customer receives a new notification. When they click the badge, a list of new and read notifications is displayed.

Each notification in the list consists of a severity level icon, an icon that indicates the origin/category of the notification, a title, a body, and a creation date. The notifications in the list are sorted by date with the newest on top. The developer can choose between an exact date and a friendlier label, for example, Now, 59 Min ago, or yesterday. This is configurable in widget preferences.

Severity levels include the following:

  • Success

  • Information

  • Warning

  • Alert

The severity level does not have impact on the sorting of notifications by date.

The icons indicating the origin of a notification include the following, powered by respective DBS capabilities:

  • Transactions

  • Messages

  • Payments

  • Limits

  • Contacts

  • Saving Goals (Powered by Personal Finance Management capability)

  • Financial Insights (Powered by Personal Finance Management capability)

  • Account Statements

The developer sets up widget preferences to display all notifications or only those that have certain origins and severity levels. For example, only the notifications coming from Limits and/or only those that have the severity level of Warning. Multiple badges can be used in an experience, displaying notifications from different origins or severity levels. This preference must be set in a consistent manner with the View sticky and popup notifications widget.

Unread notifications are highlighted. The panel at the top of the list indicates the total number of notifications.

The bank customer can delete a notification by clicking the delete icon in the detail view of that notification.

The customer clicks the icon to access the Manage business approval notification preferences widget. Developers can hide the icon using widget preferences.

A developer can customize navigation from certain types of notifications in the list. In this case, when the bank customer clicks the notification, they are redirected to the corresponding widget. This includes the following:

Notification type Destination widget

New message received

View conversation thread by the customer

Contact change rejected or approved

View contact list

Payment rejected or approved

Manage payments

See Configure navigation for Notifications widgets.

The bank customer selects a notification in the list to view its details, and then returns to the notification list.

The number of lines after which a notification is truncated, as well as the presence of a Show more button that opens truncated notification details, is configurable in widget preferences.

Features

This widget has the following features for the bank customer:

  • View the count of unread notifications.

  • View the notifications in a list.

  • View the details of a single notification.

  • Click through to the corresponding widget.

  • Mark all notifications as read.

  • Go to notification settings.

  • Delete notifications.

View sticky and pop-up notifications

In the View sticky and pop-up notifications widget, the bank customer or employee receives alerts about important events.

The alerts can be of two types:

  • Sticky: A sticky banner is displayed at the top of the screen whenever a new notification is received. These alerts are suitable for important messages that have to attract attention. For example, scheduled maintenance, changes in work hours, etc.

  • Pop-up: A pop-up is displayed in the corner of the screen whenever a new notification is received. These alerts are suitable for quick updates, such as incoming messages.

If several notifications are received, they are stacked on top of each other, with the newest notification on top. The notification disappears upon expiration or when the customer or employee dismisses it. Upon expiration or dismissal, the notification appears in the list of all notifications. If dismissed, the notification is marked as read in the list of all notifications.

Each notification consists of a severity level icon, a title, and a body. The creator of the notification text may also choose to include a link in the body. The number of lines after which a notification is truncated, as well as the presence of a Show more button that opens truncated notification details, is configurable in widget preferences.

Severity levels include the following:

  • Success

  • Information

  • Warning

  • Alert

Pop-up notifications also include an icon that indicates the origin/category of the notification. These include the following, powered by respective DBS capabilities:

  • Transactions

  • Messages

  • Payments

  • Limits

  • Contacts

  • Saving Goals (Powered by Personal Finance Management capability)

  • Financial Insights (Powered by Personal Finance Management capability)

  • Account Statements

The developer sets up widget preferences to display all notifications or only those that have certain origins and severity levels. For example, only the notifications coming from Limits and/or only those that have the severity level of Warning. This preference must be set up in a consistent manner with the Notification badge widget.

A developer can customize navigation from certain types of notifications in the list. In this case, when the bank customer clicks the notification, they are redirected to the corresponding widget. This includes the following:

Notification type Destination widget

New message received

View conversation thread by the customer

Contact change rejected or approved

View contact list

Payment rejected or approved

Manage payments

See Configure navigation for Notifications widgets.

The number of lines after which a notification is truncated, as well as the presence of a Show more button that opens truncated notification details, is configurable in widget preferences.

Features

This widget has the following features for the bank customer or employee:

  • View new notifications.

  • Dismiss a notification.

  • View notification details by clicking a link in the body (if any).

Manage business approval notification preferences

In the Manage business approval notification preferences widget, the bank customer sets up notifications for changes in the approval status. Specifically, the customer can get notifications for either or all of the following events:

  • Another user rejects a payment submitted for approval by the logged-in bank customer.

  • Another user rejects the changes to contacts submitted for approval by the logged-in bank customer.

  • Another user submits changes to contacts that the logged-in customer needs to approve.

  • Another user creates a new payment that the logged-in customer needs to approve. Notifications can be sent either immediately or at a specified time.

The Business Banking Web App - USA Edition uses transfers and payees to denote payments and contacts, respectively.

The customer toggles the switch to enable or turn off notifications. The widget saves the changes made to the notification preferences or displays an error message if they can’t be saved.

This widget interfaces with the Actions DBS capability using the action-client-api.

Features

This widget provides the following features for the bank customer:

  • View available approval notifications.

  • Enable or turn off approval notifications.

  • Receive notifications about payment approvals either in real time or grouped, at a specified time.
