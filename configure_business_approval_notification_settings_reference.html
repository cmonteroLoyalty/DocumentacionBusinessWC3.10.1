Selector name

This widget has the following selector name: <bb-actions-business-approval-notifications-settings-widget-ang>

Required imports

The following imports are required to set up and customize this widget:

  • Angular Common module import

    import { CommonModule } from '@angular/common';

  • Backbase Core module import

    import { BackbaseCoreModule } from '@backbase/foundation-ang/core';

  • Widget module import

    import { ActionsBusinessApprovalNotificationsSettingsWidgetModule } from '@backbase/actions-business-approval-notifications-settings-widget-ang';

Extension slots

This widget has the following extension slots:

  1. bbActionsToggleRecipeFormCustomizableDirective

  2. bbActionsBusinessApprovalNotificationsSettingsWidgetCustomizableDirective

  3. bbActionsBusinessApprovalListCustomizableDirective

  4. bbActionsBusinessApprovalWarningNotificationHeaderCustomizableDirective

  5. bbActionsBusinessApprovalSuccessNotificationHeaderCustomizableDirective

  6. bbActionsScheduleNotificationsComponentCustomizableDirective

bbActionsToggleRecipeFormCustomizableDirective

This extension slot includes the following default template:

<ng-template bbActionsToggleRecipeFormCustomizableDirective let-hostRef>
  <ng-container *ngIf="hostRef.formGroup">
    <form [formGroup]="hostRef.formGroup">
      <div class="bb-block bb-block--md bb-stack">
        <div
          class="bb-stack__item bb-text-support bb-text-bold"
          [attr.data-role]="'toggle-recipe-label-' + hostRef.specification.id"
        >
          {{ hostRef.label }}
        </div>
        <bb-switch-with-loading
          [isLoading]="hostRef.isLoading"
          class="bb-stack__item bb-stack__item--push-right"
          formControlName="active"
          (change)="hostRef.onSubmit(hostRef.formGroup.value)"
          aria-label="{{ 'Switch on/off ' + hostRef.label }}"
          i18n-aria-label="Toggle recipe aria label@@actions-common.actions-toggle-recipe-form.switcher.aria-label"
          [attr.data-role]="'toggle-recipe-' + hostRef.specification.id"
        ></bb-switch-with-loading>
      </div>
      <bb-actions-schedule-notifications
        class="bb-stack__item"
        *ngIf="hostRef.scheduleEnabled && hostRef.formGroup.value?.active && hostRef.actionRecipe"
        [scheduleForm]="hostRef.formGroup.get('recurrence')"
        [actionRecipe]="hostRef.actionRecipe"
        (submit)="hostRef.onScheduleChange($event)"
      ></bb-actions-schedule-notifications>
    </form>
  </ng-container>
</ng-template>

Access Control

A permission is the combination of a business function and a single privilege granted to a user on a given resource type. Privileges are enforced by Digital Banking Services. Widget behavior is affected by the permissions of the user accessing the widget.

The following table describes how the widget behavior changes depending on the permissions of the user:

Resource

Business function

Privilege

Widget behavior

Actions

Manage Action Recipes

view

The user can view notification settings.

If the user does not have the permission to view, the widget will display an empty list.

execute

The user can execute a notification setting.

If the user does not have permission to execute, the widget will display an error when a submission is made.

If the user does not have permission to approve, the widget will display an error when a submission is made.

delete

The user can delete a notification setting.

If the user does not have permission to delete, the widget will display an error when a submission is made.

create

The user can create a notification setting.

If the user does not have permission to create, the widget will display an error when a submission is made.

edit

The user can edit a notification setting.

If the user does not have permission to edit, the widget will display an error when a submission is made.
