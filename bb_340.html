Business Banking Widget Collection v.3.4.0

Highlights

The highlights for this release are:

  • All widgets of this collection are upgraded to Angular 8.

  • This widget collection now uses Foundation Angular v4.2.2.

See the release notes for each capability in Widget Collection Business Banking:

  • Account Statements - no changes.

  • Authentication - no changes.

  • Batches - no changes.

  • Contacts v3.4.0

  • Payments v3.4.0

  • Products v3.4.0

  • Sweeping - no changes.

  • Transactions - no changes.

Contacts v3.4.0

New features

The new features in this release are:

  • Contact Manager widget

    • Create a contact and add associated account(s).

    • Search for a contact.

    • Delete a contact.

Knowns issues

The following known issues have been introduced in this release:

  • If you are using Contacts in conjunction with DBS and Entitlements, and Approvals is enabled, the following features do not work:

    • Success and failure notifications when the user deletes a contact.

    • User notification that deletion is pending for approval.

Payments v3.4.0

New features

The new features in this release are:

  • Initiate Business Payment

    • New widget in Widget Collection Business Banking that focuses on specific payment flows.

    • The payments review screen is optimized for business banking.

Products v3.4.0

New features

The new features in this release are:

  • Account Picker

    • New widget with the following features:

      • Search

      • Select multiple accounts

      • Group accounts by favorite

  • Account Details

    • New widget with the following features:

      • Show account details by ID

  • You can now add custom product kinds to your widgets without using DBS.

    To add a custom product kind, create a custom Angular service using the RAML specs for the //product-summary-presentation-spec/v2/productsummary endpoint. For example:

  • PRSUM-4753 - Cannot navigate to Account Details widget from Accounts Overview card view.

  • PRSUM-4742 - Favorite Accounts always needs to be shown on top in Account Picker.
