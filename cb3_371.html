Widget Collection Business Banking v3.7.1

The highlights for this release are:

  • Capabilities

    • Account statements v3.7.1

      • Improved widget reference documentation

    • Contacts v3.7.1

      • Bug fixes

    • Notifications v3.7.1

      • New widget preferences to filter notifications by origin and severity level

    • Payments v3.7.1

      • View list of payment templates

      • Show transfer amount when selecting a payment template

  • Migration

To find out which Backbase product versions have been verified together, see Backbase product compatibility.

Account statements v3.7.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Improved documentation for View and download business account statements widget

    The following additional documentation has been added to the reference section of the widget:

    • Imports for components used inside the extension slots

    • Imports of the modules the widget is dependent on

    The widget can be wrapped into a custom one by simply following the steps added in the reference section.

Bug fixes

This release includes the following bug fixes:

  • INFREP-159: Length of "Date Range" input box is too long

    The length of the Date Range input box is now displayed in the correct size. This improves the user experience when filtering in the View and download business account statements widget.

Contacts v3.7.1

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-10110: Unable to customize notification messages in contact manager

    Notification messages for Contact Create, Contact Details, and Contact Edit widgets were not customizable.

  • MAINT-10319: Notifications are not visible when an extension slot is used

    When customizing extension slots for Contact Manager My Requests and Contact Manager Pending Approvals widgets, notification messages were set to blank.

  • MAINT-10376: Edit contact form doesn’t have an access to server error

    In extension slot bbContactEditCustomizable of the bb-contact-edit-widget there was a 'No access to the server' error.

Notifications v3.7.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Filtering notifications by origin and severity level

    The Notification badge and View sticky and pop-up notifications widgets now include preferences that allow filtering of notifications by their origin and severity level. This allows the bank developer to set up the widgets so that only the matching notifications are shown to the customer in the notification badge, list, and popup. For example, only the notifications that have the severity level of Warning and/or the origin of Payments. If these preferences are not set, the widgets display all notifications.

    Set up preferences for both widgets in a consistent manner, especially if you are using multiple badges to display different kinds of notifications in the experience.

Bug fixes

This release includes the following bug fixes:

  • MAINT-10255: The bb-notifications-badge-widget has hardcoded IDs

Payments v3.7.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • View list of payment templates

    Now it is possible to view list of all allowed payment templates in one place. This overview of templates is accessible via new menu option 'Templates' in out of the box offering.

  • Show transfer amount when selecting a payment template

    Now it is also possible to decide a suitable payment template, based upon transfer amount in it. The feature to 'Search select and apply of a payment template' also shows transfer amount as additional information.

Bug fixes

This release includes the following bug fixes:

  • MAINT-10315: An aria-label of the Initiate omni payment needs improvement

    The Initiate omni payment (bb-payord-omni-payment-widget-ang) has a Select a template dropdown with an aria-label set to 'DropDown button'. It does not accurately mention the control or its purpose. The aria-label is updated to specify itâ€™s a 'Payment template dropdown button'.

Known issues

This release has the following known issues:

  • Discovered in v3.7.1

    • PAYM-820: Disabled approve and reject buttons is enabled in the details popup

      If a user doesn’t have a permission to approve or reject a payment, these buttons are correctly disabled for this payment in list view of payments in manage payments. But when this payment is clicked for showing payment details, user is able to see these buttons enabled.

      If user clicks on these enabled buttons, backend is correctly rejecting this action due to lack of permissions for the user.

Migration

No migration needed.
