Set up a widget development environment

This section shows you how to:

  • Set up your widget development environment

  • Run your widget on a standalone server

  • Run your widgets in an experience

Set up your widget development environment

Follow these steps to set up your widget development environment:

  1. Install NodeJS and npm

    NodeJS is a JavaScript runtime for Backbase CLI (bb-cli) tools which helps you automate common tasks during widget development.

    Download 14.x version of NodeJS from https://nodejs.org/en/download and follow the installation instructions for latest LTS version.

    By default npm (NodeJS Package Manager) is installed at the same time. Make sure that your npm version is >=6.14.13 to avoid known problems with older versions.

    If you need a newer version of npm, run the following command:

    npm install -g npm@^6.14.13

  2. Add npm user

    Run the following command in your shell. When prompted, provide your repo.backbase.com login credentials and email:

    npm adduser
    npm adduser --registry=https://repo.backbase.com/artifactory/api/npm/npm-backbase/ --always-auth --scope=@backbase

    This will set your npm client to get all the packages belonging to @backbase scope from Backbase registry.

    As a result of this command you can expect your npm configuration file: ~/.npmrc (On Windows: %USERPROFILE%/.npmrc) to have following content:

    npmrc file content
    @backbase:registry=https://repo.backbase.com/artifactory/api/npm/npm-backbase/
//repo.backbase.com/artifactory/api/npm/npm-backbase/:_authToken=<YOUR_UNIQUE_AUTHENTICATION_TOKEN>

    If you get an EACCES error when trying to install, see Fixing npm permissions.

  3. Install the angular-cli

    The Angular CLI is a command-line interface tool that you use to initialize, develop, scaffold, and maintain Angular applications.

    npm install -g @angular/cli@11.1.0

  4. Install the Backbase schematics

    @bb-cli/schematics is package used for scaffolding Backbase Widget Architecture 3 files, you will need access to Backbase private npm repo, before running.

    npm install -g @bb-cli/schematics@5.x
    Note
    		 If your project uses other Backbase packages such as data-ang, retail-ang, wealth-ang, or universal-ang, ensure they are at a version compliant with the version of schematics that you install. To check the correct version of schematics for earlier versions of Foundation Angular, see the specific documentation for that version.

Run your widget on a standalone server

Follow these steps to test your widget on a standalone server:

  1. Scaffold a new project

    Use Angular CLI in combination with Backbase schematics:

    ng new my-example-project --collection=@bb-cli/schematics

    By default Backbase dependencies, like @backbase/foundation-ang, will be retrieved from Backbase’s registry. If you’re hosting Backbase dependencies on your own registry you should use the --registry option.

    This creates the following file structure in my-example-project:

    my-example-project
    ├─ .editorconfig
    ├─ .git
    ├─ .gitignore
    ├─ .npmrc
    ├─ .prettierrc
    ├─ README.md
    ├─ angular.json
    ├─ apps
    ├─ libs
    ├─ node_modules
    │    └─ ...
    ├─ package-lock.json
    ├─ package.json
    ├─ tscoinfig.json
    └─ tslint.json.xml

    This is a basic project structure.

    In the response you are warned to install peer dependencies. For example:

    npm WARN @backbase/foundation-ang@3.33.0 requires a peer of rxjs@6.0.0 but none is installed. You must install peer dependencies yourself.
npm WARN @backbase/data-http-ang@1.52.5 requires a peer of @backbase/core-ang@^2.50.1 but none is installed. You must install peer dependencies yourself.
npm WARN @backbase/tab-container-ang@1.12.1 requires a peer of rxjs@~6.0.0 but none is installed. You must install peer dependencies yourself.
npm WARN @backbase/column-container-ang@1.12.1 requires a peer of rxjs@~6.0.0 but none is installed. You must install peer dependencies yourself.
npm WARN @backbase/panel-container-ang@1.12.1 requires a peer of rxjs@~6.0.0 but none is installed. You must install peer dependencies yourself.
npm WARN ajv-keywords@3.4.0 requires a peer of ajv@^6.9.1 but none is installed. You must install peer dependencies yourself.
npm WARN axe-webdriverjs@0.2.0 requires a peer of selenium-webdriver@^2.46.1 but none is installed. You must install peer dependencies yourself.

    Update them all manually. For example:

    npm install rxjs --save

  2. Add an application container and a widget

    In order to start using your project, you must add at least one application container and one widget to the basic project structure. For example:

    cd my-example-project
npm run ng -- generate app --name=my-example-app

    This scaffolds an app inside the apps directory of your project.

  3. Install the Backbase theme

  4. Add a widget

    Add an existing or new widget to your project:

    • Install an existing widget

    • Scaffold a new one:

      npm run ng -- generate widget --name=example-widget --npmScope=example-widget

      The npmScope option is compulsory if no scope has been provided during scaffolding the project or thereafter.

  5. Test your application on a standalone server

    npm start
