Access Control

A permission is the combination of a business function and a single privilege granted to a user on a given resource type. Privileges are enforced by Digital Banking Services. Widget behavior is affected by the permissions of the user accessing the widget.

The following table describes how the widget behavior changes depending on the permissions of the user:

Resource

Business function

Privilege

Widget behavior

Payments

SEPA CT

view

The current user views a list of payments.

If the user does not have permission to view, the widget will display an empty list.

approve

The current user is able to approve or reject a listed payment.

If the user does not have permission to approve, the buttons to approve and reject will be unavailable.

cancel

The current user is able to cancel a listed payment awaiting approval.

If the user does not have permission to cancel, the button to cancel a payment will be unavailable.

US Domestic Wire

view

The current user views a list of payments.

If the user does not have permission to view, the widget will display an empty list.

approve

The current user is able to approve or reject a listed payment.

If the user does not have permission to approve, the buttons to approve and reject will be unavailable.

cancel

The current user is able to cancel a listed payment awaiting approval.

If the user does not have permission to cancel, the button to cancel a payment will be unavailable.

Custom Payment

view

The current user views a list of payments.

If the user does not have permission to view, the widget will display an empty list.

approve

The current user is able to approve or reject a listed payment.

If the user does not have permission to approve, the buttons to approve and reject will be unavailable.

cancel

The current user is able to cancel a listed payment awaiting approval.

If the user does not have permission to cancel, the button to cancel a payment will be unavailable.
