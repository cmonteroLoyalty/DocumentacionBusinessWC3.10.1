Access Control

A permission is the combination of a business function and a single privilege granted to a user on a given resource type. Privileges are enforced by Digital Banking Services. Widget behavior is affected by the permissions of the user accessing the widget.

The following table describes how the widget behavior changes depending on the permissions of the user:

Resource

Business function

Privilege

Widget behavior

Contacts

Contacts

view

The current user views the details of listed saved contacts.

If the user does not have the permission to view, the widget shows an empty container.

bbContactDetailsCustomizable

This extension slot includes the following default template:

<ng-template bbContactDetailsCustomizable let-hostRef>
  <div *ngIf="!(hostRef.isDeleting | async); else deleting">
    <div *ngIf="(hostRef.contact | async) as currentContact; else loading">
        <bb-alert-ui modifier="warning"
          *ngIf="hostRef.isPendingApproval(currentContact)"
          title="An approval request for this contact is open"
          i18n-title="@@contact-details.alert.pending-approvals.title"
          message="Further modification is not possible on the contact until the approver takes action"
          i18n-message="@@contact-details.alert.pending-approvals.message"
        >
        </bb-alert-ui>
        <div
          class="bb-block d-xl-none bb-block--lg"
          *ngIf="hostRef.showBackToListMobile$ | async"
        >
        <button
          bbButton
          color="unstyled"
          class="bb-text-bold text-primary"
          (click)="hostRef.gotoList()"
        >
          <bb-icon-ui name="arrow-back"></bb-icon-ui>
          <ng-container i18n="@@contact-manager-business.details.backToList.btn.label">
            Back to My Contacts
          </ng-container>
        </button>
      </div>
      <bb-contact-details-contact-information
        [contact]="currentContact"
        [isPendingApproval]="hostRef.isPendingApproval(currentContact)"
        (delete)="hostRef.onRequestDelete()"
        (edit)="hostRef.onEdit()"
      ></bb-contact-details-contact-information>

      <bb-confirm-action-dialog
        [isOpen]="hostRef.isDeleteConfirmActionDialogOpen"
        [options]="{ contact: currentContact }"
        [confirmButtonColor]="'danger'"
        (confirm)="hostRef.onConfirmDelete($event.contact)"
        (cancel)="hostRef.onCancelDelete()"
      >
        <span
          title
          i18n="@@contact-details.confirm-action-dialog.title"
          id="title"
        >
          Delete contact?
        </span>
        <span
          data-role="delete-contact-confirmation-message"
          body
        >
          <span i18n="@@contact-details.confirm-action-dialog.body-start">
           Are you sure you want to delete the contact
          </span>
          <span class="bb-text-bold">
            "{{currentContact.name}}"?
          </span>
          <span i18n="@@contact-details.confirm-action-dialog.body-end">
            This action can not be undone.
          </span>
        </span>
        <span
          cancelButtonLabel
          i18n="@@contact-details.confirm-action-dialog.button.cancel"
        >
          Not now
        </span>
        <span
          confirmButtonLabel
          i18n="@@contact-details.confirm-action-dialog.button.confirm"
        >
          Delete
        </span>
      </bb-confirm-action-dialog>
    </div>
  </div>
</ng-template>
