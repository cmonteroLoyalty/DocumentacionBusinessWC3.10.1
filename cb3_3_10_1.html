To find out which Backbase product versions have been verified together, see Backbase product compatibility.

Batches v3.10.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Ability to translate and show transaction codes to transaction descriptions and account types for ACH batches

    It is now possible to display a transaction code description and account type from a payment transaction code information within the batch details view.

    This functionality is configurable per batch type and has been preconfigured for the ACH Credit and ACH Debit batch types.

Notifications v3.10.1

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-12256: Retail notifications widget not displaying correctly on small devices

    The mobile screen resolution issue is now fixed.

Payments v3.10.1

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • PAYM-2135: Sending arrangement id for selected own account as counterparty

    Currently for a selected own account as beneficiary, its account number and scheme is sent in counterparty details of a payment. It is incorrect behaviour which is also blocking for upcoming changes of masked own account numbers. This has been fixed in both Initiate omni payment and Initiate payment wizard, where arrangement id and scheme id of the respective selected own account is sent in counterparty details of a payment.

    Note: This could impact for any custom payment validations within BE which are done based upon account number and scheme/format for a counterparty, only if such validations are done before the OOTB validation of this own account, for being present in arrangements and accessible to this user. This OOTB arrangements validation already works with arrangement id and additionally retrieves account number and scheme/format of this arrangement. Hence such custom validations can be done after this validation using these retrieved account number details.

Products v3.10.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Support and configuration for user and account alias

    • Display name key

      In the previous release, DBS introduced a new displayName key that holds the label of the account to show to the user. The arrangement-manager service supports multiple methods to set the value for the displayName key:

      • User alias (existing): Alias that is strictly personal.

      • Account alias (new): Alias that applies to all users with access to the account.

        The account alias applies to all users with access to the account. For example, for company administrators who want all financial employees to see the same account alias.

      See DBS: Display name and account aliases and DBS: Products v2.21.0 release notes.

    • App-level configuration for user or account alias

      This release, developers can configure the web app to use the user alias or the account alias.

    • Display and edit the user or account alias in the View business account details widget

      Depending on the app-level configuration, the View business account details widget shows the user or account alias.

      If the account alias is configured, users with the edit privilege for the Manage Arrangement Alias business function are allowed to edit the account alias to help their organisation distinguish accounts from each other.

    • Displaying the alias

      If set, the account or user alias propagates across the following widgets:

    • View business account list

    • View favorite accounts

    • Select business account

  • FedWire routing number visible in View business account details

    If populated, the bankBranchCode2 key is visible in the View business account details widget as FedWire Routing Number.

Transactions v3.10.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Discard a dispute or inquiry report about a transaction

    Bank customers now see a confirmation modal when discarding a dispute or inquiry report about a transaction. This prevents the bank customer to accidentally discard a dispute or inquiry report.

This release note explains changes to this capability.

Bug fixes

This patch includes the following bug fix:

  • MAINT-12542: Conflicting data-ang version dependency

    To prevent errors during installation, the widgets in this capability have been updated to remove bad peer dependencies.
