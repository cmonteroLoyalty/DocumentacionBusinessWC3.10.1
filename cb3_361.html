Account statements v3.6.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Custom date ranges for filtering

    The customer can now easily filter statements in the View and download business account statements widget using predefined date ranges. These ranges include day, week, month, quarter, and year.

  • Pagination added as a preference

    A new paginationType preference has been added to the View and download business account statements widget. Using this preference, the frontend developer can select between pagination and a Load more button for the widget.

Authentication v3.6.1

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-9389: Unable to use custom provider tokens for the Log in user widget

    To resolve this issue, providers are now added at the module level instead of the component level.

Batches v3.6.1-sp.1

This release note explains changes in this patch.

Bug fixes

This patch includes the following bug fix:

  • CB-4970: Limit breach details do not show on Firefox and Safari browsers

    When a user tries to approve a batch which results in a limit breach, the widget presents a notification informing the user that the batch breaches a limit. The user is able to select the details button within the notification, which results in a modal opening containing additional information on the breached limit. However, this modal is not opening on the browsers Firefox and Safari.

Batches v3.6.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Ability to upload balanced ACH credit batch payment files (NACHA) - Beta (sample code)

    The Upload batch payments widget is extended with the functionality to upload balanced ACH credit batch files (NACHA). .ACH files can now be uploaded, and are be visible in My uploads upon successful upload. The user is notified of success or failure of their upload. Each batch is seperately manageable in the Manage batch payments widget.

    This is a beta (sample code) release. This feature may, unless otherwise authorized by Backbase, be used for testing purposes only and is provided on an "AS IS" basis without support of any kind. You should not use this feature in a production environment.

Known issues

This release has the following known issues:

  • Discovered in v3.6.1

    • CB-4970: Limit breach details do not show on Firefox and Safari browsers

      When a user tries to approve a batch which results in a limit breach, the widget presents a notification informing the user that the batch breaches a limit. The user is able to select the details button within the notification, which results in a modal opening containing additional information on the breached limit. However, this modal is not opening on the browsers Firefox and Safari.

Cash flow forecasting v3.6.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Link accounting software

    The Manage forecast settings widget displays the flow to connect user profile in the accounting software to the Cash Flow Forecasting by using Codat, a third party software to aggregate the financial data from the online accounting systems.

  • View receivables, manage payables

    The View receivables and Manage payables widgets display a list of available receivables of an SME, based on the information retrieved from the accounting software.

  • Search or filter receivables and payables

    Search for receivables by customer name and reference number. This is implemented as per Business Banking UX patterns.

Contacts v3.6.1

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • CB-4818: No search box after contact search returns no result.

    When a user searches for a contact, and the search returns no results, a no contacts message displays briefly before the search box disappears. When the user switches between the contacts tab or menu items, the search box appears again. This has been fixed to show the search box even if there are no results.

  • CB-4837: Contacts search box uses Theme 1.

    Contacts search box has been migrated to Theme 2.

  • MAINT-9609: WC3 Contacts - NullInjectorError: No provider for ContactsService when customizing CreateContact slot.

    When trying to customize the CreateContact slot, the compiler asked to import the ContactsService service which was not exposed by the contacts commons. This made the widget not customizable. The following error was received: "NullInjectorError: No provider for ContactsService!"

    The ContactsService is now exposed, and the error resolved.

Notifications v3.6.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Notification origin icon

    Notifications in the Manage and count notifications and View sticky and pop-up notifications widgets now include an icon that indicates from which functionality they originated. For example, Messages, Payments, or Transactions.

Payments v3.6.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Filter payment orders

    The Manage payments widget now includes a filter tool. The filter uses payment order attributes in its search criteria, allowing the customer to quickly isolate desired payment orders. The filter allows a user to select one or more statuses, a minimum or a maximum amount or both, and a from or to execution date or both.

Bug fixes

This release includes the following bug fixes:

  • MAINT-9635: PayordInitiateBusinessPaymentWidget propagates an @Output named editPaymentCompleted that was not implemented.

    The editPaymentCompleted preference, needed for internal navigation, was declared in model.xml but not implemented in the widget component, which broke the widget’s customization build. This issue is now fixed.

Known issues

This release has the following known issues:

  • Discovered in v3.6.0

    • MAINT-9720: Cannot have two omni-payment widgets with different payment types.

      When two or more widgets customize the Initiate omni-payment widget and use the withConfig method to provide the payment configuration, the second or last configuration made overrides all the previous configurations.

      To work around this issue, provide the paymentConfig and the business function as inputs to the Initiate omni-payment widget component, in the wrapper widget’s template. Do not use the withConfig method. This issue is fixed with the 3.6.2 release.
