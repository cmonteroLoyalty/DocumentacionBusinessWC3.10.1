Account statements v3.8.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Use out-of-the-box Select business account widget to filter statements in View and download business account statements widget

    The View and download business account statements widget now uses the out-of-the-box Select business account widget, instead of the embedded bb-account-statement-account-selector component. This change is necessary because the out-of-the-box selector supports multi-account selection. The embedded bb-account-statement-account-selector component is deprecated and will be removed in 3.9.0.

  • Web: Prevent multiple user clicks on download statement option

    The download progress of a statement is now visible in the browser for the View and download business account statements widget in order to prevent the user clicking the download option multiple times while the download process is already initiated.

Bug fixes

This release includes the following bug fixes:

  • Accessibility issues in the View and download business account statements widget

    The screen reader does not read book date, account name, and account number. Also, the screen reader does not read the Preview and Download options on user click. These issues have been fixed.

Deprecations

This release includes the following deprecations:

  • In v3.9.0:

    • Support ceases for bb-account-statement-account-selector component. This is replaced with the out-of-the-box Select business account widget.

Notifications v3.8.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: New notification settings for contact and payment approval notifications

    New notification settings have been added to the Manage business approval notification preferences widget for contact changes and payments submitted for approval.

  • Web: Navigation from the payment approval notification

    The notification in the Notification badge and View sticky and pop-up notifications widgets about a payment submitted for approval is now clickable and redirects the recipient to payments. See functional documentation.

  • Web: Navigation from the contact approval notification

    The notification in the Notification badge and View sticky and pop-up notifications widgets about a contact change submitted for approval is now clickable and redirects the recipient to contacts. See functional documentation.

Payments v3.8.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: New designs for payment review screens

    The payment review screens of omni payment and payment wizard have the following changes:

    • A new card layout and icons for From account and Transfer to account details.

    • The balance of the From account is shown in its card.

    • The bank branch code of beneficiary’s bank is shown in Transfer to card.

    • For international transfers, the flag for the destination country of payment is shown in Transfer to card.

    • Available details such as the chosen payment option, transfer fee, and fee sharing option display in the review screen.

  • Web: Show payment details of international transfer in Manage payments

    Manage payments is adopted to show details of a international transfer. This includes showing additional international transfer specific details such as:

    • Currency rates

    • Converted amount

    • Transfer fee

    • Fee sharing option

    • Flag for destination country of payment

  • Web: Show details of a payment template in Manage payment templates

  • Manage payment templates now has the functionality to show details of templates selected from the list.

  • Web: Delete a payment template in Manage payment templates

    Manage payment templates now has the functionality to delete a payment template from the list.

  • Web: Initiate omni payment config for International Transfers out of beta

    The config for International Transfers in the Initiate omni payment widget is now out of beta and fully supported.

  • Web: Payment wizard out of beta

    The Payment wizard widget is now out of beta and fully supported.

This release note explains changes to this capability.

CVEs

Backbase products contain open source libraries and frameworks, also known as third-party components. Information on a security vulnerability found in a third-party component is made publicly available. Vulnerabilities are typically assigned an official CVE number by MITRE. There are several publicly accessible vulnerability databases that provide background information on the vulnerability. The CVE records can also include mitigation or exploitation steps.
