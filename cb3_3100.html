All capabilities v3.10.0

This release note lists the changes to all capabilities in this release.

New features

This release provides the following new features:

  • Support for Angular 11

    All Angular widgets, journeys, and libraries have been upgraded to Angular 11.

    The widgets in this collection now support both Angular 10 and 11.

Deprecations

This release includes the following deprecations:

  • In the February 2022 release:

    • Support ceases for Angular 10.

Breaking changes

This release includes the following breaking changes:

  • The use of non-prefixed ApiModule, BASE_PATH, and ConfigurationParameters in the data-ang project is no longer supported.

To update an existing deployment to accommodate this breaking change, see Migration.

Batches v3.10.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Search, select, and apply a batch payment template

    When creating a manual batch, you can now search, select, and apply a batch template by clicking on the new Template button. After applying the template, you can edit the payment details before submitting the batch.

  • Upload a NACHA unbalanced ACH credit file

    The Upload batch payment widget now supports uploading a NACHA unbalanced ACH credit file in addition to the balanced ACH file that is already supported.

  • Access Control-driven enablement of buttons and options

    The Upload batch payment widget uses the business functions and privileges of the user to disable or enable the following buttons and options:

    • New batch button: Enabled when the user has create privilege for at least one batch type

    • Manual batch: Enabled when the user has create privilege for a manual batch type

    • Upload: Enabled when the user has create privilege for a file-based batch type

    Which business functions and privileges the widget uses to drive this enablement logic is configurable.

Notifications v3.10.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Group notifications about payment approvals

    Users can now choose to receive notifications about payments they need to approve either in the real time or grouped, several times a day. The timezone for the user notification preferences should be set in the action-service configuration.

Payments v3.10.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Adapt intended action for payment templates

    The Manage payment templates widget retrieves payment templates with the intendedAction query parameter set to is VIEW. This returns all payment templates the user is allowed to view. Additionally, the button for creating a new payment from a retrieved payment template in the list and details view is only visible when the actions key is CREATE for this template. This means the user is allowed to create a payment from this template.

    The Initiate omni payment and Initiate payment wizard widgets retrieve payment templates with the intendedAction query parameter set to is CREATE. This returns all payment templates the user is allowed to create a payment from.

  • Closed payment restrictions

    The Initiate omni payment widget is adapted to enforce closed payment restrictions. If a user only has closed payment permissions for a payment or payment template they are creating or editing, the widget restricts the user to only select a known contact or arrangement. This change is configurable via the checkClosedPaymentsAccess property, which defaults to false.

  • Omni payment full support for SEPA configuration

    SEPA configuration for omni payments is now out of beta and fully supported by the Initiate omni payment widget.

Bug fixes

This release includes the following bug fixes:

  • MAINT-11774: Manage Payments: Load more refreshes the whole list

    When clicking load more in the Manage payments widget, the whole list is refreshed instead of just the new items being appended to the bottom. This is been fixed by showing the over-list loading indicator only when there is pagination with page numbers. In case of pagination with load more button, a spinner is introduced within this button.

  • MAINT-10279: Design System - bb-alert-ui aria-describedby does not exist

    bb-alert-ui generates an ID to use for its content, and places this ID in the aria-describedby, but then does not place the ID as an actual ID on the content element. Consequently, the aria-describedby points to an element that does not exist. This has been fixed with dynamic attaching/detaching of aria properties based on content provided into the component.

Known issues

This release has the following known issues:

  • Discovered in v<3.10.0>

    • PAYM-<2139>: <Manage Payments: Notifications are not shown when approve/reject the payment>

      When approved or rejected, user is not able to see notification for success or failure of this action. This is since the timeout configuration notificationDismissTime is changed from seconds to milliseconds.

      This configuration property notificationDismissTime needs to be re-configured to desired timeout value in milliseconds.

Products v3.10.0

This release note explains changes to this capability.

Breaking changes

This release includes the following breaking changes:

  • Client-side masking

    The widgets no longer mask the following account numbers on the client side:

    • Card number: number

    • IBAN: IBAN

    • BBAN: BBAN

    The frontend relies only on backend masking. To enable masking, please configure the following correctly:

    • Digital Banking Services

      • Configure masking for product attributes

      • Products reference

    • Business Banking Widget Collection

      • Configure account numbers formatting for Products widgets

Transactions v3.10.0

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Dispute or inquire about a transaction

    Bank customers can dispute a transaction they disagree with, or inquire about a transaction by sending a secure message to their bank. This enables the bank customer to send a dispute request directly, and easily contact customer service. To initiate a dispute or an inquiry, the bank customer selects the Report button in the transaction details of the View transactions table widget.

Migration

When you migrate from one version of this product to another, best practice is to update incrementally through all minor and major versions in between.

Follow these steps to update an existing deployment:

  1. Update non-prefixed references in the data-ang project

    To move from the deprecated non-prefixed ApiModule, BASE_PATH, and ConfigurationParameters, follow these steps:

    1. Check the public_api.ts file in data-ang project for the correct prefix to use. For example, the Manage user security widget requires the prefix User.

    2. Update your project to use the prefix. For example, replace ApiModule, BASE_PATH, and ConfigurationParameters with UserAPIModule, USER_BASE_PATH, and UserConfigurationParameters.
