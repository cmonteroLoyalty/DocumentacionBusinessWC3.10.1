Payments v3.5.0

This release note explains the changes to this capability.

New features

The new features in this release are:

  • Manage an existing ACH debit

    The Manage ACH Debit web widget now has the functionality to cancel, delete, copy and print an existing ACH debit.

  • New: Initiate ACH Credit widget (web)

    The new Initiate ACH credit widget supports creating an ACH outgoing transfer. The widget has the following features:

    • Create an ACH credit transfer in page view

    • Select one of your own accounts or a registered contact as a beneficiary

    • Specify the Standard Entry Class applicable to your transfer

    • Specify the Recipient ID (ACH) of the beneficiary, if it is known already

    • Review and submit the ACH credit transfer

  • Print details of a payment order

    In the Manage Payments web widget, you can print the details of a selected payment order, including the approval log.

  • Approve or reject multiple payment orders

    In the Manage Payments web widget, you can select multiple payment orders, and approve or reject in one action. This new feature supports:

    • Select or deselect multiple payment orders on a page

    • Select or deselect all payment orders on a page

    • Select multiple payment orders across multiple pages

    • Clear your selection with one click

    • View the number of selected payments

    • Approve or reject all the selected payment orders with one action

Bug fixes

The following issues have been fixed in this release:

  • MAINT-8077: Payment orders filtered by date return in a shifted range

Deprecations

The following features are deprecated from this release:

  • Support for these features ceases in v3.6.0.

    • The following keys are deprecated in the payloads of the payments presentation specification:

      Deprecated key

      New key

      debtor/debtorAccount

      originator/originatorAccount

      creditTransferTransactionInformation

      transferTransactionInformation

      creditTransferTransactionInformation.creditor

      transferTransactionInformation.counterparty

      creditTransferTransactionInformation.creditorAccount

      transferTransactionInformation.counterpartyAccount

      creditTransferTransactionInformation.creditorBank

      transferTransactionInformation.counterpartyBank

      As a consequence, the following widgets will stop supporting the deprecated keys in payment orders, and will migrate to the new keys in v3.6.0:

      • Initiate Payment

      • Initiate Business Payment

      • Manage Payments
