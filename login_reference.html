Log in user

Preferences

Title and description Name Editable for Default value

Login Success Redirect To

Set the URI pattern to configure the redirect URL. The widget redirects the user to this URL when they log in successfully.

If you configure this preference, do not set a value in the Output Login Action preference.

loginSuccessRedirectTo

Admin

mpa:/index

Output Login Action

Set the action to perform after successful login.

If you configure this preference, remove the value from the Login Success Redirect To preference.

output.loginAction

Admin

Access Control

The user does not require any permissions to access this widget’s functionality.

Extension slots

This widget has the following extension slots:

  1. bbIamLoginCustomizable

  2. bbIamLoginFormCustomizable

  3. bbIamLoginAddtionalFieldsCustomizable

  4. bbIamLoginFormHeaderCustomizable

bbIamLoginCustomizable

Configure this extension slot as follows:

<ng-template bbIamLoginCustomizable let-hostRef>
  <bb-login-form (loginSuccess)="hostRef.onLoginSuccess()"></bb-login-form>
</ng-template>

Login redirect URL

To replace the redirect URL for a login request, follow these steps:

  1. Create a new provider by either adding it to the module class or creating a new file.

  2. Configure the provider as follows:

    1. Set the apiRoot to the desired login URL.

    2. Reference the injectable BB_LOGIN_HEADER_CONFIG_TOKEN. This replaces the contents of any other configuration that uses this string.

      export const LoginConfigCustomProvider: Array<Provider> = [{
  provide: BB_LOGIN_HEADER_CONFIG_TOKEN,
  useValue: {
    Accept: '*/*',
    'Content-Type': 'application/x-www-form-urlencoded',
    'Cache-Control': 'no-cache, no-store, must-revalidate',
  },
}, {
  provide: BB_LOGIN_URL_CONFIG_TOKEN,
  useValue: {
    baseUri: '',
    gatewayRoot: "/gateway",
    apiRoot: 'https://example.newroute.com/gateway/api',
    authUri: '/auth',
    loginUri: '/login',
  }
}]

  3. Import the new provider into the app.module.ts or the login-widget.module.ts module.

  4. Add the provider to the providers array.

    @NgModule({
  imports: [
    CommonModule,
    ReactiveFormsModule,
    HttpClientModule,
    BackbaseCoreModule.withConfig({
      classMap: { LoginWidgetComponent },
    }),
    FocusModule,
  ],
  declarations: [...components, ...customizables],
  exports: [...components, ...customizables],
  providers: [... LoginCustomFormProvider, ...],
})

  5. Reimport your config providers. The new provider is injected and replaces the previous one.

    The widget then uses the redirect URL specified in the apiRoot.

Note
 This widget does not support mock providers, because it does not use a data module. Instead, the widget sends a request to the login URL directly through the HTTP client. If you have not configured an authentication service in your local development environment, the widget throws an HTTP 404 Not Found error.

Output login action

To perform an action after successful login, follow these steps:

  1. Set up the action for the widget to perform

    For example, if a user is assigned to only one service agreement, you can set the widget to select the context for the user.

    Either:

    • Configure an output handler

      • To redirect the user to another page, use the URL output handler. This takes the value /<experience>/index and appends it to the base URL. Alternatively, you can provide the entire URL, for example, mywebsite.com/homepage.

      • To perform an action instead of a redirect, use the Web SDK to develop your own output handler.

    • Wrap a widget extension

      1. Extend the login widget as needed.

      2. Include an output for loginAction on the login-widget element:

        <bb-login-widget-ang (loginAction)="performExtraAction()"></bb-login-widget-ang>

        This enables the widget to perform an action after login.

      3. Provide your own mechanism for performing regular page navigation. For example, configure outputting and use the URL output handler as above.

  2. Configure the Log in user widget preferences

    Configure the Log in user widget preferences as follows:

    1. Remove the value from the loginSuccessRedirectTo preference.

    2. Configure the output.loginAction preference to hook into the action using the output handler, or leave it blank if performing a widget extension.
